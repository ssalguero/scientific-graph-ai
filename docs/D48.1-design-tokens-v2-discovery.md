## 3. Baseline — sistemas actuales

```text
src/lib/ui/
  tokens.ts     — primitives (spacing, radius, shadows, transitions, animation, zIndex, elevation)
  theme.ts      — semantic class strings + helpers (shell, panels, buttons, sidebar, alerts, …)
  icons.ts      — UI_ICONS + getIcon
  index.ts      — barrel @/lib/ui

src/components/workspace/
  WorkspaceTokens.ts — WORKSPACE_TOKENS { shell, mainColumn, inner }

src/components/ui/
  buttons/*     — consume getButtonVariant / theme button classes
  layout/*      — consume getPanelStyle / panel tokens
  sidebar/*     — consume sidebar* theme tokens + getIcon

src/app/projectFileUiStyles.ts — re-exports project-file button/field variants from theme (compat)
```

### 3.1 Responsabilidades por sistema

| Sistema | Path | Responsabilidad |
|---------|------|-----------------|
| **Primitive tokens** | `src/lib/ui/tokens.ts` | Fragmentos Tailwind reutilizables (spacing, radius, shadows, transitions, animation, zIndex, elevation). Sin semántica de producto. |
| **Theme / semantic classes** | `src/lib/ui/theme.ts` | Class strings canónicas + `getAppShell` / `getButtonVariant` / `getPanelStyle` / `getBadgeStyle` / `getStatusColor`. Incluye CSS variables `--app-*` vía `appShellLight` / `appShellDark`. |
| **Icon registry** | `src/lib/ui/icons.ts` | Mapa semántico → glyph (emoji/ASCII). Sin librerías de iconos. |
| **Workspace shell tokens** | `src/components/workspace/WorkspaceTokens.ts` | Layout shell post-D47: `shell`, `mainColumn`, `inner`. Congelados; ownership del Workspace. |
| **Sidebar tokens** | `theme.ts` (`sidebar*`) | Anchos, shells, overlay, nav states, rail chrome (D46). Consumidos por `components/ui/sidebar`. |
| **Button tokens** | `theme.ts` (`btn*`, `actionBar*`, helpers) | Variantes de botón + action bar. Consumidos por `components/ui/buttons` y `page.tsx`. |
| **Panel tokens** | `theme.ts` (`card`, `contentPanel`, `emptyState`, …) + `getPanelStyle` | Superficies / paneles de contenido. Consumidos por layout UI + page. |

### 3.2 Relación con `themeMode`

- `themeMode: "light" | "dark"` (app-preferences)
- `getAppShell(themeMode)` aplica `appShellLight` / `appShellDark` (variables `--app-*`)
- `WorkspaceLayout` compone `WORKSPACE_TOKENS.shell` + `getAppShell(themeMode)`
- Sidebar / buttons / panels usan `var(--app-*)` — compatibles con el shell

## 4. Inventario (Token → Destino D48)

| Token / grupo | Ubicación | Uso actual | Consumidores principales | Destino en D48 |
|---------------|-----------|------------|--------------------------|----------------|
| **Layout (app shell CSS vars)** | `theme.ts` `appShellLight` / `appShellDark` / `getAppShell` | Fondo + `--app-*` por modo | `WorkspaceLayout`, theme consumers | Consolidar bajo API tokens v2 **sin** cambiar valores visuales |
| **Layout (workspace shell)** | `WORKSPACE_TOKENS.shell` | `flex min-h-screen flex-col lg:flex-row` | `WorkspaceLayout` | Preservar freeze; opcional re-export desde UI tokens **aditivo** |
| **Layout (main column)** | `WORKSPACE_TOKENS.mainColumn` | Scroll column | `WorkspaceContent` | Idem |
| **Layout (inner)** | `WORKSPACE_TOKENS.inner` | Padding + `space-y-3` | `WorkspaceContent` | Idem; evaluar dedupe vs `spacing` primitives |
| **Spacing** | `tokens.ts` `spacing` | px/py/gap/space-y/mb/mt fragments | Compuesto en theme / futuro wiring | Centralizar; reducir duplicación con strings inline restantes |
| **Radius** | `tokens.ts` `radius` | rounded-md/lg/xl/full | theme compositions | Mantener; ampliar uso donde haya duplicados |
| **Typography** | Disperso en `theme.ts` (`panelHeading`, `sectionLabel`, `fieldLabel`, …) | Tamaños / pesos / tracking | page, panels, sidebar | Inventariar y agrupar bajo tokens tipográficos v2 (move-only) |
| **Elevation / Shadows** | `tokens.ts` `shadows` + `elevation` | shadow-sm/md, interactive | sidebar overlay, cards, buttons | Unificar naming; eliminar overlap shadows↔elevation si aplica |
| **Borders** | Embebido en theme (`border-[var(--app-border)]`, dashed, etc.) | Contornos de paneles / inputs | page, layout, sidebar | Extraer fragmentos reutilizables sin cambio visual |
| **Transitions / Animation** | `tokens.ts` `transitions` + `animation` | duration-200/300, active scale, grid collapse | sidebar, buttons, shell | Mantener; tema solo vía composición |
| **zIndex** | `tokens.ts` `zIndex` | stacking overlay/modal/toast | sidebar overlay/trigger | Mantener |
| **Sidebar** | `theme.ts` `sidebar*` | Shell, overlay, nav states, widths | `components/ui/sidebar` | **Compatible**; no breaking; consolidación aditiva |
| **Buttons** | `theme.ts` `btn*` / `actionBar*` / `getButtonVariant` | Variantes Primary/Outline/Danger/… | `components/ui/buttons`, page | **Compatible**; dedupe con action bar si hay overlap |
| **Workspace** | `WorkspaceTokens.ts` | Shell layout only | `components/workspace` | Freeze D47 vigente; D48 no romper API |
| **Panels** | `theme.ts` panels + `getPanelStyle` | card/content/empty/results | `components/ui/layout`, page | Preparar D50 panels sin rediseño |
| **Icons** | `icons.ts` `UI_ICONS` / `getIcon` | Glyphs semánticos | sidebar, buttons | Freeze público; sin librerías nuevas en D48 |
| **Alerts / toggles / data / results** | `theme.ts` | Estados UI científicos | page + paneles | Consolidar naming; fuera de rediseño |

### 4.1 Observación de deuda (solo inventario)

- Primitivos en `tokens.ts` **no** cubren tipografía ni borders como grupos de primer nivel.
- Muchas class strings semánticas viven solo en `theme.ts` (composiciones largas).
- Posible overlap: `shadows` vs `elevation`; spacing primitives vs `WORKSPACE_TOKENS.inner`.
- `page.tsx` aún consume muchas constantes de `theme` directamente (esperado; D48 consolida, no rediseña).

D48.2 abordará consolidación **move-only** de estas deudas sin cambiar apariencia.

## 6. Objetivos Design Tokens v2

| Objetivo | Descripción |
|----------|-------------|
| Centralización completa | Una jerarquía clara: primitives → semantic theme → domain shells (workspace/sidebar) |
| Eliminación de duplicación | Misma clase / fragmento no definida en N sitios |
| Sistema escalable | Extensible para D49–D52 sin reabrir shell |
| Compatibilidad `themeMode` | `getAppShell` + `--app-*` preservados |
| Compatibilidad Workspace | `WORKSPACE_TOKENS` freeze D47 intacto (aditivo only) |
| Compatibilidad Sidebar | `sidebar*` + nav states D46 intactos |
| Compatibilidad paneles futuros | `getPanelStyle` / panel tokens listos para D50 |

Nombre documental de la superficie consolidada: **`UI_TOKENS`** (ver API Freeze). Puede materializarse como re-export/agrupación sobre `tokens.ts` + selecciones de `theme.ts` — detalle de forma en D48.2, **sin** breaking changes.

## 7. API Freeze (D48)

### 7.1 Superficies congeladas

| Superficie | Contrato |
|------------|----------|
| **`UI_TOKENS`** | Agrupación pública prevista de primitivos (+ tipografía/borders si se extraen move-only). Forma exacta se fija en D48.2 **sin** romper exports existentes de `spacing` / `radius` / … |
| **`WORKSPACE_TOKENS`** | `{ shell, mainColumn, inner }` — freeze D47; keys y valores exactos |
| **Theme helpers públicos** | `getAppShell`, `getButtonVariant`, `getPanelStyle`, `getBadgeStyle`, `getStatusColor` + class exports actuales de `@/lib/ui/theme` |
| **Icon registry público** | `UI_ICONS`, `getIcon`, `UiIconName` |

### 7.2 Regla

```text
No breaking changes permitted during D48.
```

- Renombres / remociones de exports públicos → Amendment
- Cambios de valores de clase que alteren look → fuera de alcance
- Extensiones **aditivas** permitidas (nuevas keys, re-exports)
