## 7. Workspace API Freeze

A partir de D47.1 se congela la superficie pública prevista del módulo `src/components/workspace/` (implementación en D47.2).

### 7.1 Props congeladas

| Tipo | Campos públicos congelados (contrato) |
|------|----------------------------------------|
| `WorkspaceLayoutProps` | `themeMode`, `sidebar`, content tree (hijo `WorkspaceContent`), `panels`, `className?` |
| `WorkspaceContentProps` | `toolbar`, `workspace` |
| `WorkspacePanelsProps` | `children` (contenido overlay; hoy toasts) |

### 7.2 Tokens congelados

```ts
// Contrato documental — NO implementado en D47.1
export const WORKSPACE_TOKENS = {
  shell: "flex min-h-screen flex-col lg:flex-row",
  mainColumn: "flex-1 min-w-0 overflow-auto",
  inner: "w-full px-3 sm:px-4 lg:px-5 xl:px-6 py-2.5 sm:py-3 space-y-3",
} as const;
```

`getAppShell(themeMode)` permanece en `@/lib/ui/theme` y se **compone** en `WorkspaceLayout` (no es un nuevo sistema de design tokens; eso es D48).

### 7.3 Regla de freeze

```text
No breaking changes permitted during D47.
```

Cambios a nombres de props, forma de `WORKSPACE_TOKENS`, o exports del barrel durante D47 requieren Amendment explícito. D48+ debe ser **aditivo** respecto a esta superficie.

## 8. Slot API (documental — no implementada)

API pública conceptual para D47.2+:

```text
WorkspaceLayout
├── sidebar          → <Sidebar />
├── (content child)  → <WorkspaceContent />
│     ├── toolbar    → header + tabs + session chrome
│     └── workspace  → scientific sections + alerts
└── panels           → <WorkspacePanels> toasts… </WorkspacePanels>
```

Notas:

- El slot de contenido científico se documenta como **`workspace`** (no como `children` genérico en el contrato público), aunque React pueda mapearlo internamente a `children`.
- `panels` es el slot de overlays; hoy solo toasts.
