## 6. Gestión de Amendments

Operacionaliza D38.3 §7–§8 **sin modificar el procedimiento original**.

### 6.1 Cuándo Amendment vs Resolution Note

| Situación | Instrumento |
|-----------|-------------|
| Cambio de scope · restricciones · ownership · roadmap oficial · criterios de gates · API Freeze | **Amendment D38.x** (D38.3 §7) |
| Aclarar interpretación **sin ampliar scope** | **Resolution Note** `RN-D38.x` (D38.3 §8) |
| Edición silenciosa de D38.* OFFICIAL | **Prohibida** |

### 6.2 Información mínima requerida

**Amendment** (D38.3 §7): ID · Motivo · Secciones afectadas · Diff documental · Impacto · Aprobación.

**Resolution Note** (D38.3 §8): ID · Ambigüedad · Resolución · Referencia · Límite (sin ampliar scope).

### 6.3 Flujo de aprobación (operativo)

| Paso | Acción |
|------|--------|
| 1 | Redactar instrumento con campos mínimos |
| 2 | Evaluar impacto vs D38.2 / D38.4 / D38.5 / API Freeze · riesgos D39.3 |
| 3 | Aprobación explícita registrada |
| 4 | Append en Governance Decision Log (D38.3 §9) |
| 5 | Solo entonces aplicar el cambio normativo autorizado |

### 6.4 Evidencia documental

Instrumento emitido · entrada Decision Log · referencia en acta de épica / microfase afectada.

## 8. Gestión de excepciones

Procedimiento operativo consistente con D38.3 §6:

| Paso | Acción operativa |
|------|------------------|
| 1 | **Identificar** desviación vs D38.2 / D38.4 / API Freeze |
| 2 | **Documentar** evidencia (sección afectada · impacto · riesgo D39.3 si aplica) |
| 3 | **Evaluar impacto** (scope · épicas · gates · API) |
| 4 | **Decidir** Resolution Note (solo aclaración) **o** Amendment |
| 5 | **Registrar** en Governance Decision Log |
| 6 | **Continuar** solo tras instrumento aprobado; **detener** ejecución si hay bypass o FAIL de gate |

Prohibido bypass silencioso (D38.3 §6 paso 5).
