## 2. Metodología de rúbrica

Escala **1–5** en todos los ejes, interpretada como **aptitud para inaugurar la primera épica BUILD** (5 = más favorable).

| Eje | 5 significa | Ancla evidencia |
|-----|-------------|-----------------|
| Complejidad técnica | Baja complejidad de arranque | Tamaño estimado D37.2 §7 · superficie API Freeze |
| Acoplamiento | Bajo acoplamiento a monolito/SCI | D37.1 matriz · handlers vs SCI-40 |
| Riesgo de regresión | Bajo riesgo sobre GRAPH certificado | Gates prod2e · barrels D29–D35 · B-05 estable |
| Impacto funcional v1.0 | Alto valor de producto/cierre | Carry-in EXPORT-1-01/02 · hueco post-GRAPH |
| Reducción potencial LOC | Alta reducción en `page.tsx` | B-15 / B-16 / B-17 |
| Reutilización | Alta reutilización transversal | Cobertura tipos VGB / engines existentes |
| Gobernanza | Alta encaje gates/API Freeze/estabilidad | VGB-R1 · sin bump schema · IN-TREE vs UNGOVERNED |

**Total:** suma de 7 ejes (máximo **35**).

**Desempate:** (1) impacto v1.0 · (2) riesgo de regresión · (3) gobernanza.

**Umbral objetivo de cambio** vs candidato preferente al cierre D36 (EXPORT-1): otro candidato solo desplaza la hipótesis si supera a EXPORT-1 por **≥3 puntos** totales **y** gana al menos **4 de 7** ejes.

Ninguna métrica nueva de código; ninguna re-medición fuera de citas literales D37.1/D37.2.
