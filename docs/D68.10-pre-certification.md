## 1. Resumen ejecutivo

**D68 — Session Autosave Foundation** entrega la infraestructura desacoplada de **persist-on-mutate inteligente** para Session Registry: dirty tracking + debounce + flush vía Bridge D66, sin UI, sin Recovery / `SessionSnapshot` (D69) y sin Workspace Restore (D70).

Componentes implementados bajo `src/components/session/autosave/` + wiring privado en `SessionProvider`. Compatible con Freezes D65 (Session), D66 (Persistence) y D67 (Restore) **sin refactor** de Registry, Persistence API, Restore Engine ni Context.

Documentación oficial actualizada. Suite `validate:d68-gate` en verde. Proyecto listo para **D68.11 — Certification (CA-D68)**.

---

## 2. Arquitectura final

```text
SessionProvider (owner privado)
        │
        │ notifyMutation after successful API mutation
        ▼
SessionAutosaveController
        │
        ├─ DirtyTracker
        ├─ AutosaveScheduler
        └─ AutosaveFlushPolicy
                │
                ▼
SessionPersistenceBridge (D66)
        │
        ├─ persistSession / persistRegistry
        └─ (unregister path) SessionStorageAdapter.clear → persistRegistry
                │
                ▼
IndexedDB ScientificGraphAI / sessions (D66 Adapter)
```

**SessionRegistry permanece el SSOT.** Autosave solo observa mutaciones ya aplicadas y escribe vía Bridge.
