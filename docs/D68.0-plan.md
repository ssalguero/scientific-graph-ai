## 1. Resumen ejecutivo

**D68 — Session Autosave Foundation** introduce la infraestructura desacoplada de **persist-on-mutate inteligente** para Session Registry: dirty tracking + debounce + flush vía Bridge D66, sin alterar Session (D65), Persistence (D66) ni Restore (D67).

Al cierre de D68 existirá un módulo independiente bajo `src/components/session/autosave/` con:

- contratos Freeze (`AutosaveMutationKind`, `AutosaveMutationEvent`, `SessionAutosaveController`, Options, `AUTOSAVE_DEBOUNCE_MS`)
- `DirtyTracker` (ids dirty + `needsFullRewrite` solo vía unregister)
- `AutosaveScheduler` (debounce)
- `AutosaveFlushPolicy` (persistSession vs persistRegistry vs clear+persistRegistry)
- `SessionAutosaveController` (`notifyMutation` / `pause` / `resume` / `flush` / `dispose`)
- wiring privado en `SessionProvider` (sin exponer Autosave en Context)
- barrel público exclusivo `@/components/session/autosave`
- suite de validadores + certificación

**SessionRegistry permanece el SSOT.** Autosave solo observa mutaciones ya aplicadas y escribe vía Bridge. No hay UI, no hay Recovery / `SessionSnapshot` (D69), no hay Workspace Restore (D70).

Espejo de responsabilidades:

| Serie | Dirección |
|-------|-----------|
| **D66** | Registry → serialize → Adapter.save |
| **D67** | Adapter.load → records → deserialize → Registry.register |
| **D68** | Mutación → política → Bridge write |

```text
Session mutation (Provider API)
        → DirtyTracker + Scheduler (D68)
        → SessionAutosaveController.flush
        → bridge.persistSession | persistRegistry [| adapter.clear + persistRegistry]
        → Serializer → Adapter → IndexedDB (D66, sin cambios)
```

## 4. Arquitectura

### 4.1 Ubicación del módulo (previsto D68.1+)

```text
src/components/session/autosave/
    AutosaveTypes.ts
    DirtyTracker.ts
    AutosaveScheduler.ts
    AutosaveFlushPolicy.ts
    SessionAutosaveController.ts
    index.ts
```

**Reglas de colocación:**

- **Nunca** dentro de `windows/`
- **Nunca** mezclado en el núcleo del Session Registry (archivos D65 top-level)
- Módulo **completamente separado** bajo `session/autosave/`
- **No** se re-exporta desde `src/components/session/index.ts`
- Entrada pública exclusiva: `@/components/session/autosave`
- **No** crear carpetas adicionales bajo `autosave/`

### 4.2 Separación de responsabilidades

| Componente | Responsabilidad única | No hace |
|------------|----------------------|---------|
| **AutosaveTypes** | Contratos Freeze | I/O, React, lógica dirty/debounce |
| **DirtyTracker** | Ids dirty + `needsFullRewrite` (**solo** vía unregister) | Persist, debounce; setear rewrite en updateState |
| **AutosaveScheduler** | Debounce / cancel / flush-now | Conocer Bridge / IndexedDB |
| **AutosaveFlushPolicy** | Decidir operación de flush | Mutar Registry; clear sin flag unregister |
| **SessionAutosaveController** | `notifyMutation` → dirty → (si !paused) schedule → flush; concurrencia B | Restore, UI, Context; flush implícito en dispose |
| **SessionProvider** (D68.6) | Ownership privado + `notifyMutation` tras mutación exitosa | Exponer Autosave en Context |
| **autosave/index.ts** | Única exportación pública del módulo | Re-export Persistence / Restore / Session barrel |

### 4.3 Pipeline certificado

```text
Provider API mutation OK
        ↓
controller.notifyMutation(event)
        ↓
DirtyTracker (mark / markRemoved)
        ↓
AutosaveScheduler (debounce) — skipped while paused
        ↓
AutosaveFlushPolicy(snapshot)
        ↓
persistSession | persistRegistry | (clear + persistRegistry)
        ↓
IndexedDB ScientificGraphAI / sessions (D66 Adapter)
```

Nota: la rama `clear` **solo** si `needsFullRewrite` (unregister real). `updateState` / `register` nunca la toman.

### 4.4 Diagrama de dependencias

```text
SessionProvider (owner privado)
        │
        │ notifyMutation after successful API mutation
        ▼
┌──────────────────────────────┐
│ SessionAutosaveController    │
│  DirtyTracker · Scheduler    │
│  FlushPolicy                 │
└────────────┬─────────────────┘
             │
             ├─► SessionPersistenceBridge.persistSession / persistRegistry
             └─► SessionStorageAdapter.clear  (solo needsFullRewrite)
                      │
                      ▼
                 IndexedDB (D66 sole I/O)
```

### 4.5 Integración Provider (mínima, D68.6 — no en D68.0)

1. Crear `SessionAutosaveController` una vez (`useRef`) con Bridge + Adapter.
2. Tras `register` / `unregister` / `updateState` **exitosos**, `notifyMutation`.
3. En unmount: `dispose()` (cancel timers · **sin** flush implícito). Durabilidad = `await flush()` previo si el caller lo requiere.
4. **No** cambiar `SessionBridge.tsx` (sigue Window → `windowIds`; autosave se dispara porque Bridge ya llama `api.updateState`).
5. **No** cambiar `page.tsx` / WindowManager / UI.
6. Actualizar/superseder `d66.bound.provider.noAutoPersistIo` para permitir llamadas **solo** vía Controller.

Bootstrap default-session: el register one-shot de mount **no** dispara autosave (`HR-no-default-mount-persist`).

### 4.6 Semántica Freeze — clear / pause / concurrencia / dispose

#### A) `clear()` solo tras unregister real

`adapter.clear()` borra **toda** la store `sessions` y luego `persistRegistry()` reescribe. Correcto para fidelidad post-`unregister` (Adapter sin `delete(id)`), pero **caro**.

```text
needsFullRewrite = true  ⇔  únicamente notifyMutation({ kind: "unregister", ... })
register / updateState     →  NUNCA activan needsFullRewrite
flush exitoso (ruta clear) →  limpia needsFullRewrite + dirty snapshot consumido
```

Prohibido:

```text
updateState → (flag errónea) → clear() → persistRegistry()
```

#### B) `pause()` / `resume()` — dirty sí, flush no

```text
pause()
  → notifyMutation SIEMPRE marca dirty
  → NO schedule / NO flush mientras paused
resume()
  → si hay dirty pendiente → flush() inmediato (sin esperar debounce)
```

**Prohibido:** que `pause()` ignore `notifyMutation` (se perderían mutaciones durante restore).

#### C) Flush concurrente — modelo B (congelado)

```text
flush running
  → nuevo notifyMutation → dirty se acumula
  → flush actual termina
  → si dirty o needsFullRewrite → otro flush
```

Un solo `flushInFlight`. **Sin** cola de Promises de flush (modelo A rechazado).

#### D) `dispose()` — sin flush implícito (congelado)

```text
dispose()
  → cancel timer / scheduler
  → NO flush implícito
  → Controller inutilizable (notifyMutation no-op)
```

## 5. API Freeze

Toda superficie listada aquí queda **congelada** en D68.0. Implementación (D68.1+) debe respetarla exactamente. **Sin implementación en D68.0.**

### 5.1 AUTOSAVE_DEBOUNCE_MS

```ts
const AUTOSAVE_DEBOUNCE_MS = 1000 as const;
```

### 5.2 AutosaveMutationKind / AutosaveMutationEvent

```ts
type AutosaveMutationKind = "register" | "unregister" | "updateState";

type AutosaveMutationEvent = {
  readonly kind: AutosaveMutationKind;
  readonly sessionId: SessionId;
};
```

### 5.3 SessionAutosaveController

```ts
type SessionAutosaveController = {
  /** Always updates dirty (even while paused). Schedules flush only if !paused && !disposed. */
  notifyMutation(event: AutosaveMutationEvent): void;
  /** Suppress schedule/flush; dirty tracking continues. */
  pause(): void;
  /** Clear paused; if dirty pending → flush() immediately (no debounce wait). */
  resume(): void;
  /** Single-flight: concurrent calls coalesce via dirty-remain + re-flush (model B). */
  flush(): Promise<void>;
  /** Cancel timer only. NO implicit flush. Subsequent notifyMutation is no-op. */
  dispose(): void;
};
```

### 5.4 Options + factory

```ts
type SessionAutosaveControllerOptions = {
  readonly debounceMs?: number; // default = AUTOSAVE_DEBOUNCE_MS (1000)
};

createSessionAutosaveController(
  bridge: SessionPersistenceBridge,
  adapter: SessionStorageAdapter,
  options?: SessionAutosaveControllerOptions
): SessionAutosaveController;
```

### 5.5 Superficies internas de módulo (no barrel — D68.2+)

**DirtyTracker:** `mark(id)` · `markRemoved(id)` · `clear()` · `snapshot()` → `{ dirtyIds; needsFullRewrite }`.

- `mark(id)` — register / updateState → dirty ids only; **no** toca `needsFullRewrite`
- `markRemoved(id)` — unregister → dirty + `needsFullRewrite = true` (**única** vía)

**Scheduler:** `schedule(cb)` · `cancel()` · `flushNow()` · `dispose()`.

**FlushPolicy (puro):**

| Condición | Operación | Nota |
|-----------|-----------|------|
| `needsFullRewrite === true` | `clear` + `persistRegistry` | Solo si flag vino de unregister real |
| `needsFullRewrite === false` ∧ 1 dirty id | `persistSession(id)` | Nunca clear |
| `needsFullRewrite === false` ∧ 2+ dirty ids | `persistRegistry()` | Nunca clear |
| vacío | no-op | — |

### 5.6 Barrel público (`autosave/index.ts`)

**Allowlist:**

- `AUTOSAVE_DEBOUNCE_MS`
- `AutosaveMutationKind` · `AutosaveMutationEvent`
- `SessionAutosaveController` · `SessionAutosaveControllerOptions`
- `createSessionAutosaveController`

**Prohibido:**

- Re-export desde `session/index.ts`
- Constantes IndexedDB
- Re-exports de persistence / restore
- Hooks React / status UI
- APIs de DirtyTracker / Scheduler / FlushPolicy (internas)

### 5.7 Compatibilidad D65 / D66 / D67 (intacta)

| Superficie | Acción D68 |
|------------|------------|
| `SessionDefinition` / `SessionState` / `SessionRegistry` | **Sin modificación** |
| `SessionPersistenceRecord` / Adapter / Bridge / Serializer / Deserializer | **Sin modificación** |
| `restore/*` | **Sin modificación** · Autosave **no** importa restore |

| `session/index.ts` | **Sin** re-export de `autosave/*` |
| `SessionContext` / `SessionAPI` | **Sin** Persistence ni Autosave |

## 6. Hard Rules

| ID | Regla |
|----|-------|
| **HR-autosave-bridge-only** | Escritura de persistencia solo vía Bridge Freeze (`persistSession` / `persistRegistry`). |
| **HR-autosave-adapter-clear-only** | Único uso directo de Adapter en Autosave = `clear()` en ruta unregister-fidelity. Sin `save` / `load` desde Autosave. |
| **HR-clear-only-on-unregister** | `clear()` **solo** cuando `needsFullRewrite` fue set por `kind === "unregister"`. `updateState` / `register` nunca disparan clear. |
| **HR-no-persistence-api-change** | Prohibido modificar Record / Adapter API / Bridge API / Serializer / Deserializer. |
| **HR-no-restore-change** | Prohibido modificar Restore Engine / Types / Validator / Report. |
| **HR-no-registry-change** | Prohibido modificar `SessionRegistry` ni sus semánticas. |
| **HR-registry-ssot** | Registry sigue siendo el único SSOT runtime; Autosave solo observa mutaciones ya aplicadas. |
| **HR-context-no-autosave** | Context / API no exponen Autosave, dirty, flush ni Bridge. |
| **HR-barrel-isolation** | `session/index.ts` **must NOT** export `autosave/*`. |
| **HR-no-ui** | Sin UI / indicadores / botones / toasts en D68. |
| **HR-no-react-in-core** | DirtyTracker / Scheduler / FlushPolicy / Types = puro TS (sin React). Wiring React solo en Provider. |
| **HR-pause-dirty-continue** | En `pause()`, `notifyMutation` **sigue** marcando dirty; **no** schedule / flush. `resume()` hace flush si hay dirty. |
| **HR-pause-during-restore** | Durante restore batch el Controller debe estar `pause()`d (contrato); Auto Restore UI wiring = futuro. |
| **HR-flush-single-flight** | Un solo flush in-flight. Mutaciones durante flush acumulan dirty; al completar → re-flush si dirty (modelo B; sin cola). |
| **HR-dispose-no-implicit-flush** | `dispose()` cancela timer y **no** hace flush. Durabilidad = `await flush()` previo explícito. |
| **HR-no-default-mount-persist** | El register one-shot de default session **no** dispara autosave. |
| **HR-debounce-default** | Default debounce = 1000ms Freeze. |
| **HR-import-isolation** | `autosave/*` no importa windows / tabs / content / series / layout / toolbar / inspector / UI ni `restore/*` ni project draft autosave. |
| **HR-no-project-draft-coupling** | Aislado de `useProjectDraftAutosave` / store de proyectos. |
| **HR-no-snapshots** | Sin `SessionSnapshot` / Recovery (D69). |
| **HR-no-workspace-restore** | Sin restore Window / Tab / Content / Series (D70 / Workspace Persistence). |
| **HR-d65-d66-d67-compat** | Gates previos permanecen PASS; cero refactor de código certificado. |
