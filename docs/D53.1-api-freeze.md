---

## 1. Alcance del Freeze

Este freeze cubre **únicamente** la capa de interacción Dock.

| Incluido en este freeze | Excluido (permanecen en D51/D52 freeze) |
|-------------------------|-----------------------------------------|
| `DockInteractionProvider` | `DockProvider` / `DockContext` / `useDockContext` |
| `DockInteractionContext` | `DockState` v1 |
| `useDockInteraction` · `useDockDrag` · `useDockResize` | Registry / Registration / Visibility |
| `DockInteractionState` + session types | Layout / Slots / `DOCK_FEATURES` |
| Public interaction methods | Hosts D51 (`DockRoot` / `DockZone` / `DockPanel` contracts base) |

Extensiones posteriores a D53 deben ser **aditivas** sobre estos contratos. Cambiar firmas o eliminar campos = **breaking** = **prohibido hasta D60 inclusive**.

| Símbolo | Rol congelado |

## 5. State & Types

### 5.1 DockInteractionState

Campos congelados:

| Campo | Tipo conceptual | Semántica |
|-------|-----------------|-----------|
| `focusedDock` | `string \| null` | Foco teclado / a11y |
| `activeDock` | `string \| null` | Única ventana activa UI |
| `hoverDock` | `string \| null` | Último dock bajo pointer; `null` al salir |
| `draggingDock` | `string \| null` | Id en drag (espejo de `dragSession?.dockId`) |
| `resizingDock` | `string \| null` | Id en resize |
| `dragSession` | `DockDragSession \| null` | Payload sesión drag |
| `resizeSession` | `DockResizeSession \| null` | Payload sesión resize |

```text
hoverDock ≠ focusedDock ≠ activeDock
activeDock ≠ DockState.activePanelIds (visibility)
```

### 5.2 Vistas / aliases congelados

| Tipo | Rol |
|------|-----|
| `DockFocusState` | Vista de foco (`focusedDock` / related) |
| `DockDragState` | Vista de drag (`draggingDock` + `dragSession`) |
| `DockResizeState` | Vista de resize (`resizingDock` + `resizeSession`) |

La implementación puede materializarlos como types alias o slices; los nombres y el significado quedan congelados.

### 5.3 ResizeEdge

```ts
type ResizeEdge = "n" | "s" | "e" | "w" | "ne" | "nw" | "se" | "sw";
```

### 5.4 DockDragSession

```ts
interface DockDragSession {
  dockId: string;
  pointerId: number;
  start: { x: number; y: number };
  current: { x: number; y: number };
}
```

### 5.5 DockResizeSession

```ts
interface DockResizeSession {
  dockId: string;
  pointerId: number;
  edge: ResizeEdge;
  start: { x: number; y: number };
  current: { x: number; y: number };
}
```

## 6. Public API (firmas conceptuales)

### 6.1 Focus & Activation

| Método | Firma conceptual | Comportamiento mínimo |
|--------|------------------|------------------------|
| `focus` | `focus(id: string): void` | Set `focusedDock` |
| `blur` | `blur(): void` | Clear `focusedDock` |
| `activate` | `activate(id: string): void` | Set unique `activeDock`; implica focus |
| `deactivate` | `deactivate(): void` | Clear `activeDock` |

### 6.2 Drag session

| Método | Firma conceptual | Comportamiento mínimo |
|--------|------------------|------------------------|
| `beginDrag` | `beginDrag(...): void` | Crea `DockDragSession`; set `draggingDock` |
| `updateDrag` | `updateDrag(...): void` | Actualiza `current` |
| `endDrag` | `endDrag(): void` | Limpia sesión + `draggingDock` |

Parámetros exactos de `beginDrag` / `updateDrag` deben incluir al menos la información necesaria para poblar `DockDragSession` (`dockId`, `pointerId`, coordenadas). Detalle de overloads aditivos permitido; **remover o renombrar métodos = breaking**.

### 6.3 Resize session

| Método | Firma conceptual | Comportamiento mínimo |
|--------|------------------|------------------------|
| `beginResize` | `beginResize(...): void` | Crea `DockResizeSession`; set `resizingDock` |
| `updateResize` | `updateResize(...): void` | Actualiza `current` |
| `endResize` | `endResize(): void` | Limpia sesión + `resizingDock` |

Igual regla: must populate `edge` + session fields; no layout mutation en D53.

### 6.4 Stubs

```text
STUBS = ALLOWED IN D53
BODIES MAY BE NO-OP OR STATE-ONLY
SIGNATURES MUST NOT CHANGE THROUGH D60
NO LAYOUT / SIZE MUTATION REQUIRED IN D53
```

### 6.5 Fuera de la API pública D53

| Evento documental | API pública D53 |
|-------------------|-----------------|
| Float | **No** |
| Dock | **No** |
| Close | **No** |

Reservados a D54+ sin romper este freeze (nuevos métodos = extensión aditiva futura, no parte del contrato D53 mínimo).

## 7. Governance Freeze

### 7.1 One-way dependency

```text
DockProvider
        ↓
DockInteractionProvider
```

| Regla | Enforcement previsto |
|-------|----------------------|
| `DockInteractionProvider` **never** imports `DockProvider` | `validate-d53-governance` |
| `DockProvider` / `DockContext` **never** import interaction modules | `validate-d53-governance` |
| Children may consume `useDockContext` and/or `useDockInteraction` | Documental |
| No circular dependencies | `validate-d53-governance` |

### 7.2 Separación de estados

| Prohibido |
|-----------|
| Añadir `focusedDock` / `activeDock` / hover / sessions a `DockState` v1 |
| Mutar `DockState.sizes` desde drag/resize en D53 |
| Confundir `hoverDock` con `focusedDock` o `activeDock` |

### 7.3 Zero UX / isolation

| Prohibido en módulos interaction |
|----------------------------------|
| Imports científicos (graphs / math / statistics / analysis / export) |
| Tailwind nuevo / colores hardcodeados |
| Animaciones |
| Imports desde `page.tsx` hacia interaction API (wiring Zero UX) |
