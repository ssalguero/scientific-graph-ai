### 4.3 Umbrales oficiales EXPORT-1

**Entorno de referencia (documental):** navegador desktop Chromium-class · máquina de desarrollo típica · gráfico VGB representativo con contenido visible (`chartExportRef` con tamaño > 0). Valores exactos de hardware se registran en el informe de medición de Testing.

| ID | Dimensión | Umbral (PASS) | FAIL si… | Notas |
|----|-----------|---------------|----------|-------|
| **PT-PNG-T** | Tiempo export PNG (user-facing `exportChartPng` / captura equivalente) | ≤ **5 s** caso típico; ≤ **15 s** caso denso/complejo documentado | Excede banda sin justificación en acta Testing | Medir wall-clock captura+download trigger |
| **PT-SVG-T** | Tiempo export SVG (`exportChartSvg`) | ≤ **5 s** típico; ≤ **15 s** denso/complejo | Idem | SVG puede diferir de PNG; reportar ambos |
| **PT-MEM** | Consumo de memoria | Sin OOM del tab; pico de captura **no** provoca crash; crecimiento no ilimitado en N exports consecutivos (N≥3) | Crash OOM · degradación grave reportada | Medición best-effort (Performance/Memory API o observación); documentar método |
| **PT-DPI** | Calidad / DPI / resolución PNG | `pixelRatio` efectivo ≥ **2** (baseline) · data URL `data:image/png` válida · longitud > 100 (criterio actual de captura) | PNG vacío/inválido · downgrade no documentado de ratio | Alineado a captura existente |
| **PT-SVG-Q** | Calidad SVG | data URL SVG válida · descarga usable · sin canvas raster “vacío” como único resultado cuando el Scope exige vector | SVG inválido / vacío / error no manejado | Carry-in EXPORT-1-02 se dispone en D41.5; umbral mínimo = export funcional no roto |
| **PT-VGB9** | Gráficos certificados (9 tipos VGB) | Smoke/perf sobre **conjunto representativo** de tipos certificados; **cero** regresión grave de export en tipos ejercitados | Fallo export en tipo ejercitado sin fix | Cobertura completa 9/9 preferida; si subset, listar tipos + justificación en acta Testing |
| **PT-REG** | Regresión de performance | Sin regresión grave vs comportamiento pre-BUILD documentado (mismo escenario) | Regresión grave reportada sin remediation | D38.5 §10 PASS clause |
| **PT-ACC** | Criterio de aceptación de rendimiento (rollup) | PT-PNG-T · PT-SVG-T · PT-MEM · PT-DPI · PT-SVG-Q · PT-VGB9 · PT-REG satisfacen PASS en Testing | Cualquier PT crítico FAIL sin waiver RN/Amendment | Alimenta Performance Gate |

### 4.4 Criterios de aceptación de rendimiento (resumen)

Performance Gate (D38.5 §10) para EXPORT-1 se considera **PASS** en Testing/Certification cuando:

1. Informe de medición existe y cita umbrales **PT-*** de este documento.
2. PT-PNG-T · PT-SVG-T · PT-DPI · PT-SVG-Q · PT-MEM · PT-VGB9 · PT-REG = PASS (o waiver formal RN/Amendment).
3. No hay regresión grave de export reportada.
4. Invariantes API Freeze (D41.3) intactos en el mismo ciclo de evidencia.

**D41.4 no declara Performance Gate PASS** — solo habilita su evaluación futura.

- Objetivo: handlers PNG/SVG · captura no-null con nodo válido · no romper invariantes D41.3.

### 5.2 Smoke Tests

- Smoke export PNG/SVG sobre tipos VGB ejercitados (PT-VGB9).
- Smoke de no-regresión UI mínima (botones PNG/SVG habilitados con contenido).
- Creación de scripts smoke = solo en BUILD si hace falta · **no** en D41.4.

### 5.3 Regression

- **Obligatorio:** `validate:prod2e-gate` PASS (Regression Gate D38.5 §12).
- Sin regresión VGB/GRAPH reportada en tipos certificados.
- VGB-R1 respetado (efímeros no persistidos).
