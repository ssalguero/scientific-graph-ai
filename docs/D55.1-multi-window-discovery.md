## 1. Executive Summary

**Multi-Window Foundation** es la infraestructura base que permitirá al Workspace gestionar múltiples ventanas/paneles independientes en series posteriores, **sin** alterar todavía el comportamiento científico ni la UX existente.

| Principio | Contrato D55 |
|-----------|--------------|
| Infraestructura desacoplada | Módulo `windows/` aislado; sin acoplamiento a Dock / Layout Engine / App Router |
| Preparación para Floating Windows | Contratos y lifecycle listos para D56+ |
| Cero impacto visual | Sin CSS · sin componentes visuales · sin cambios de producto |
| Cero wiring | Sin Provider en `page.tsx` · sin consumers · sin integración Workspace |
| API pública estable | Nombres y contratos congelados antes de D55.2 |

**D55 no implementa ventanas visibles.** No hay UI de ventanas, ni floating, ni render. D55 solamente define e implementará (D55.2–D55.3) la infraestructura de registro, estado, contexto y API.

```text
D55 = ARCHITECTURE + MODEL + INFRASTRUCTURE (isolated)
D55 ≠ FLOATING WINDOWS
D55 ≠ VISIBLE WINDOWS
D55 ≠ LAYOUT PERSISTENCE
```

## 6. Frozen Data Contracts

Contratos previstos (shapes congeladas conceptualmente; TypeScript exacto en D55.2).

### 6.1 WindowDefinition

| Campo | Obligatorio | Semántica |
|-------|-------------|-----------|
| `id` | Sí | Identificador estable |
| `title` | Sí | Título lógico |
| `dockId` | No | Asociación opcional a dock/panel |
| `visible` | Sí | Visibilidad de modelo (no UI) |
| `metadata` | No | Extensión tipada como `Record<string, unknown>` para D56 (iconos, tamaño inicial, tipo, etc.) sin romper el API |

```ts
export interface WindowDefinition {
  id: string
  title: string
  dockId?: string
  visible: boolean
  metadata?: Record<string, unknown>
}
```

### 6.2 WindowState

| Campo | Semántica |
|-------|-----------|
| `windows` | Colección de definiciones (contrato público: `Map<string, WindowDefinition>`; la implementación interna puede diferir si el contrato expuesto se respeta) |
| `activeId` | Id de ventana activa (opcional) |
| `focusedId` | Id de ventana con foco (opcional) |
| `minimizedIds` | Conjunto de ids minimizados (contrato público: `Set<string>`) |

```ts
export interface WindowState {
  windows: Map<string, WindowDefinition>
  activeId?: string
  focusedId?: string
  minimizedIds: Set<string>
}
```

### 6.3 WindowEventType / WindowEvent

Eventos oficiales (conjunto cerrado):


```ts
export type WindowEventType =

export interface WindowEvent {
```

| Prohibido |
|-----------|

| resize |
| floating |
| overlays |
| portals |
| persistence |
| multi-monitor |

### 7.3 Wiring

```text
NO WIRING TO CONSUMERS

NO IMPORTS FROM windows/ INTO Dock / Sidebar / Inspector / Toolbar
```
