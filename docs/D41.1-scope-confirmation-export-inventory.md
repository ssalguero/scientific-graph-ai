## 4. Export Inventory (read-only)

**Archivo inventariado:** `src/app/page.tsx` — **sin modificación**.

**Dependencia externa observada:** `html-to-image` (`toPng`, `toSvg`) — import L13.

### 4.1 Mapa de símbolos EXPORT (PNG/SVG)

| Símbolo | Rol | Ubicación aprox. | Clasificación D41.1 |
|---------|-----|------------------|---------------------|

| `waitForChartPaint` | Doble rAF antes de captura | ~L1064 | Helper captura |
| `prepareChartExportVisibility` | Visibilidad sección/notebook para captura | ~L1079 | Helper captura |
| `restoreChartExportVisibility` | Restaura visibilidad post-captura | ~L1109 | Helper captura |
| `captureChartAsPngDataUrl` | Núcleo captura PNG (`toPng` + visibility) | ~L1133 | **Handler PNG core** |
| `chartExportRef` | `useRef<HTMLDivElement>` — nodo DOM chart | ~L15622 | Ancla captura |
| `exportChartPng` | UI → captura PNG → download | ~L15929 | **Handler PNG user-facing** |
| `exportChartSvg` | UI → `toSvg(chartExportRef)` → download | ~L15953 | **Handler SVG user-facing** |
| `ref={chartExportRef}` | Wiring DOM al contenedor chart | ~L23173 | Wiring mínimo |
| Botones PNG / SVG | `onClick={exportChartPng\|exportChartSvg}` | ~L26842–26851 | Wiring UI |

### 4.2 Observaciones de inventario (sin diseño BUILD)

1. **PNG** usa pipeline `captureChartAsPngDataUrl` + `CHART_CAPTURE_PNG_OPTIONS` (`pixelRatio: 2`).
2. **SVG** llama `toSvg` **directamente** en `exportChartSvg` (no reutiliza el helper de visibility/paint del PNG).
3. `captureChartAsPngDataUrl` también es consumido por `handleExportScientificReportPdf` (contexto `"pdf-export"`) — superficie **compartida**; PDF permanece **OUT** de EXPORT-1.
4. Baseline histórico **B-17** (~700+ LOC handlers asociados, D37.2) se **cita sin re-medición** de LOC.
