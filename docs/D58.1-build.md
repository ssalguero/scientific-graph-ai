## 1. Arquitectura implementada

```text
WindowManager
 ├── WindowState / WindowAPI          (D55 FROZEN)
 ├── WindowGeometryState              (SSOT { x, y, width, height })
 ├── WindowDragBridge                 (mutates x/y; preserves size)
 ├── WindowResizeBridge               (session only — no edge math yet)
 ├── WindowGeometryProvider
 └── WindowDragProvider
              │
FloatingWindowBridge                 (GeometryState → FloatingWindowModel[])
              │
FloatingWindowLayer
              │
FloatingWindow                       (unchanged — title-bar drag only)
```

**Pipeline (frozen; handles arrive D58.2):**

```text
Pointer → ResizeHandle → ResizeBridge → GeometryState
  → FloatingBridge → Layer → FloatingWindow
```

## 4. Decisiones

| Decisión | Motivo |
|----------|--------|
| `WindowGeometryState` (no “Store” freeze) | Storage-agnostic; Map es un backend permitido |
| Geometry unificada `{x,y,width,height}` | N/W resize necesitará posición+tamaño (D58.2) |
| ResizeBridge sin edge math | D58.1 = arquitectura; D58.2 = handles + matemática |
| `WindowResizeOrigin` congelado | `{ pointerX, pointerY, geometry }` listo para D58.2 |
| `geometry-immutable-input` | `cloneGeometry` en set/get/origin |
| Shims D57 Position* | Compat naming; autoridad = GeometryState |

## 6. Limitaciones deliberadas

- Sin resize visual / handles / pointer capture de resize
- `updateResize` no escribe GeometryState (sin edge math)
- Sin constraints / min-max / workspace clamp
- Sin `WindowResizeProvider` (llega con handles en D58.2)
- Sin validators `validate-d58-*` (D58.4)
