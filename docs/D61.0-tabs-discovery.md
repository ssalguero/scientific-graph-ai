## 1. Executive Summary

**Window Tabs Foundation** es la infraestructura que permitirá que una ventana contenga **múltiples documentos/series mediante pestañas (tabs)** de forma **completamente desacoplada** — **sin** UI, **sin** renderizado, **sin** integración visual en D61.

D61 crea **únicamente** la infraestructura (identidad, tipos, registry, selection, bridges, barrel, validators). No modifica comportamiento visible.

| Principio | Contrato D61 |
|-----------|--------------|
| Infraestructura primero | Tipos + registry + selection + mapping antes de UI |
| Desacoplamiento Window↔Tabs | Mapa externo vía `WindowTabsBridge`; sin campos tabs en Window* |
| SSOT puro Tabs | `TabRegistry` no conoce Window ni Series |
| Selection mínima | Solo `activeTab`; sin auto-política |
| Subpaquete interno | Segundo subdir de `windows/`: `tabs/` |
| Barrel único | Tabs exportable **solo** desde `windows/tabs/index.ts` |
| Zero UX / Zero React | No React · No JSX · No TSX · No hooks · No Context · No DOM · No CSS |
| Compatibilidad absoluta | WindowAPI / Series / Floating / Drag / Resize / Snap intactos |

```text
D61 = TABS INFRASTRUCTURE (identity + types + registry + selection + bridges)
D61 ≠ UI / render / visual integration
D61 ≠ Tab↔Series wiring operativo
D61 ≠ auto-select / “next tab” policy
D61 ≠ WindowAPI extension
D61 ≠ React / Context / CSS
D61 ≠ product window population
```

Equivalente incremental a:

```text
D55 → ventanas
D56 → floating
D57 → drag
D58 → resize
D59 → snap
D60 → series
D61 → tabs
```

## 7. API Semantics Freeze (LOCKED)

### 7.1 Superficies previas — LOCKED

| Superficie | Estado |
|------------|--------|
| WindowAPI D55 | **LOCKED** — sin métodos Tabs |
| WindowDefinition / WindowState | **LOCKED** — sin campos tabs (Hard Rule) |
| Series D60 | **LOCKED** — sin modificar |
| Floating / Drag / Resize / Snap | **LOCKED** |
| `windows/index.ts` | **LOCKED** — sin Tabs* |

### 7.2 TabId (LOCKED)

```ts
export type TabId = string;           // opaco — mismo patrón que SeriesId / WindowId
export function createTabId(): TabId;
export function isTabId(value: unknown): value is TabId;
```

Representación interna = `string` alias. **Prohibido** cambiar a number / branded nominal / struct en D62+.

### 7.3 TabTypes (LOCKED — no mezclar Definition con State)

```ts
/** Metadata estable — inmutable tras register (clone-on-write). */
export type TabDefinition = {
  id: TabId;
  title?: string;
  // PROHIBIDO: windowId, seriesId, TabState
};

/** Estado mutable — tipo Freeze; nunca embebido en TabDefinition. */
export type TabState = "active" | "inactive";

/** Handle mínimo. */
export type TabReference = {
  tabId: TabId;
};
```

Companion de almacenamiento en Registry:

```ts
export type TabEntry = {
  definition: TabDefinition;
  state: TabState;
};
```

### 7.4 TabRegistry (LOCKED)

SSOT exclusivo. No conoce Window ni Series. No almacena `windowId` ni `seriesId`.

Cadena:

```text
TabId → TabDefinition (estable) + TabState (mutable companion)
```

```ts
export type TabRegistry = {
  register(definition: TabDefinition): void;  // state inicial = "inactive"
  unregister(id: TabId): void;                // solo catálogo — NO toca Selection
  get(id: TabId): TabEntry | undefined;
  has(id: TabId): boolean;
  list(): readonly TabEntry[];                // INSERTION ORDER
  clear(): void;                              // no toca Selection
};
```

| Operación | Contrato congelado |
|-----------|-------------------|
| `register` | Duplicate id = no-op. Clone definition. `state = "inactive"`. |
| `unregister` | Elimina entrada. **No** limpia ni avanza `activeTab`. |
| `get` / `list` | Clone-on-read. `list()` = **orden de registro** (insertion order). |
| `clear` | Vacía catálogo. **No** toca Selection. |
| `setState` / mutators State | **Fuera** de API Freeze D61 (tipo `TabState` sí exportado). |

### 7.5 TabSelectionStore (LOCKED)

```ts
export type TabSelectionStore = {
  get(): TabId | undefined;                 // activeTab
  setActive(id: TabId | undefined): void;
  clear(): void;
};
```

- Única responsabilidad: `activeTab`.
- **NO** auto-selección.
- **NO** política “siguiente tab”.
- **NO** clear automático al `unregister`.
- Toda política de composición = **D62+**.

### 7.6 WindowTabsBridge (LOCKED)

Solo adaptación:

```ts
export type WindowId = string; // alias local — sin importar WindowTypes

export type WindowTabsBridge = {
  attach(windowId: WindowId, tabId: TabId): void;
  detach(tabId: TabId): void;
  listTabs(windowId: WindowId): readonly TabId[];  // orden de attach
  getWindow(tabId: TabId): WindowId | undefined;
  hasWindow(windowId: WindowId): boolean;
  hasTab(tabId: TabId): boolean;
  clear(): void;
};
```

- **No** estado de dominio · **No** selección · **No** lógica de negocio.
- **No** invoca Registry · **No** modifica Selection.

### 7.7 TabSelectionBridge (LOCKED)

Fachada de escritura autorizada (patrón D60):

```ts
createTabSelectionBridge(store): { get; setActive; clear }
```

Sin Window / Series / Registry.

### 7.8 Barrel Freeze (LOCKED)

| Regla | Contrato |
|-------|----------|
| Único punto público | `src/components/windows/tabs/index.ts` |
| Deep imports | **Prohibidos** desde fuera de `tabs/` |
| `windows/index.ts` | **Sin** leaks Tabs* |

Cambiar nombres Freeze, embeber State en Definition, acoplar Registry a Window/Series, o filtrar Tabs por el barrel público de `windows/` = **breaking** = **prohibido** durante D61–D63+.

## 8. Hard Rules (LOCKED)

| ID | Regla |
|----|-------|
| `HR-no-tabs-on-window-definition` | **Prohibido** agregar campos tabs a `WindowDefinition` |
| `HR-no-tabs-on-window-state` | **Prohibido** agregar campos tabs a `WindowState` |
| `HR-tabs-barrel-only` | Tabs exportable **solo** desde `src/components/windows/tabs/index.ts` |
| `HR-no-windows-barrel-leak` | **Prohibido** export Tabs* desde `windows/index.ts` |
| `HR-no-deep-imports` | **Prohibido** importar `windows/tabs/Tab*` desde fuera del barrel |
| `HR-registry-tabs-only` | TabRegistry no conoce Window ni Series; no almacena `windowId`/`seriesId` |
| `HR-definition-state-separated` | TabState nunca embebido en TabDefinition |
| `HR-no-auto-select` | Selection no auto-avanza ni limpia en `unregister` |
| `HR-bridge-mapping-only` | WindowTabsBridge solo mapeo; no Registry/Selection |
| `HR-no-touch-prior-surfaces` | No modificar WindowAPI, Series*, Floating*, Drag*, Resize*, Snap*, Layout |
| `HR-zero-ui` | Sin cambios de UI / visuales / comportamiento existente |
| `HR-tabid-string` | `TabId` permanece `string` alias |

Estas Hard Rules son **irreversibles** a efectos de D61–D63+: acoplar Tabs a `WindowDefinition`/`WindowState` o filtrar Tabs por el barrel público de `windows/` crea deuda estructural.
