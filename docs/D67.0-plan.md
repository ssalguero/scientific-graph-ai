## 1. Resumen ejecutivo

**D67 — Session Restore Foundation** introduce la infraestructura desacoplada de restauración de sesiones en memoria a partir de `SessionPersistenceRecord[]` certificados por D66, sin alterar el dominio Session (D65) ni Persistence (D66).

Al cierre de D67 existirá un módulo independiente bajo `src/components/session/restore/` con:

- contratos Freeze (`RestoreRequest`, `RestoreResult`, `RestoreStatus`, `RestoreStatistics`, errores, report)
- `RestoreValidator` puro (estructura / ids / required / metadata / duplicados — **sin** `schemaVersion`)
- `RestoreReport` data-only
- `SessionRestoreEngine` sync: `restore(request): RestoreResult`
- reconstrucción **únicamente** de `SessionRegistry` vía `deserializeRegistry` (D66) + `registry.register`
- barrel público exclusivo `@/components/session/restore`
- suite de validadores + certificación

**SessionRegistry permanece el SSOT.** Restore escribe solo a través de la API pública `register`. No hay UI, no hay auto-restore, no hay autosave, no hay `SessionSnapshot` (D69).

D67 es el **espejo inverso** de D66:

| Serie | Dirección |
|-------|-----------|
| **D66** | Registry → serialize → Adapter.save |
| **D67** | Adapter.load → records → deserialize → Registry.register |

## 4. Arquitectura

### 4.1 Ubicación del módulo (previsto D67.1+)

```text
src/components/session/restore/
    RestoreTypes.ts
    RestoreErrors.ts
    RestoreValidator.ts
    RestoreReport.ts
    SessionRestoreEngine.ts
    index.ts
```

**Reglas de colocación:**

- **Nunca** dentro de `windows/`
- **Nunca** mezclado en el núcleo del Session Registry (archivos D65 top-level)
- Módulo **completamente separado** bajo `session/restore/`
- **No** se re-exporta desde `src/components/session/index.ts`
- Entrada pública exclusiva: `@/components/session/restore`
- **No** crear carpetas adicionales bajo `restore/`

### 4.2 Pipeline certificado

```text
Caller
    ↓
SessionStorageAdapter.load()
    ↓
SessionPersistenceRecord[]
    ↓
SessionRestoreEngine.restore(request)
    ↓
RestoreValidator
    ↓
deserializeRegistry (D66)
    ↓
SessionRegistry.register(...)
    ↓
RestoreResult
```

- **I/O** (`load`) ocurre **antes** del engine (caller / futuro orquestador).
- El engine es **sync** y no conoce IndexedDB.
- `deserializeRegistry` es el deserializer **certificado en D66** — D67 no implementa un segundo mecanismo de reconstrucción.

### 4.3 Separación de responsabilidades

| Componente | Responsabilidad única | No hace |
|------------|----------------------|---------|
| **RestoreTypes** | Contratos Freeze (`Status`, `Request`, `Result`, `Statistics`) | Lógica, I/O, React |
| **RestoreErrors** | Modelos de error Freeze (data) | throw en D67.2; I/O |
| **RestoreValidator** | Validar estructura / ids / required / metadata / duplicados | Mutar records; `schemaVersion`; register |
| **RestoreReport** | Datos: warnings / errors / items / elapsed | React; persist |
| **SessionRestoreEngine** | Pipeline validate → report → deserialize → register → result | IndexedDB; UI; otros registries |
| **restore/index.ts** | Única exportación pública del módulo | Re-exportar Persistence / Session barrel |

### 4.4 Diagrama de dependencias

```text
Caller / future orchestrator
        │
        │  adapter.load()
        ▼
SessionPersistenceRecord[]
        │
        ▼
┌──────────────────────────┐
│  SessionRestoreEngine    │  sync · stateless
│  restore(request)        │
└────────────┬─────────────┘
             │
             ├─► RestoreValidator (pure)
             ├─► deserializeRegistry (D66 · pure)
             └─► SessionRegistry.register (sole write)
                      │
                      ▼
                 RestoreResult
```

## 5. API Freeze

Toda superficie listada aquí queda **congelada** en D67.0. Implementación (D67.1+) debe respetarla exactamente. **Sin implementación en D67.0.**

### 5.1 RestoreStatus

```ts
type RestoreStatus = "success" | "partial" | "failed";
```

### 5.2 RestoreRequest

```ts
type RestoreRequest = {
  readonly records: readonly SessionPersistenceRecord[];
  readonly registry: SessionRegistry;
};
```

- `registry` viaja en el request — **no** en el constructor del engine.
- Input = `SessionPersistenceRecord[]` (Freeze D66) — **no** `SessionSnapshot`.

### 5.3 RestoreStatistics

```ts
type RestoreStatistics = {
  readonly requested: number;
  readonly restored: number;
  readonly skipped: number;
  readonly failed: number;
};
```

### 5.4 RestoreResult

```ts
type RestoreResult = {
  readonly status: RestoreStatus;
  readonly report: RestoreReport;
  readonly statistics: RestoreStatistics;
  readonly restoredIds: readonly SessionId[];
};
```

### 5.5 SessionRestoreEngine

```ts
type SessionRestoreEngine = {
  /** Sync only. No overloads. No Promise. IO happens before the engine. */
  restore(request: RestoreRequest): RestoreResult;
};

createSessionRestoreEngine(): SessionRestoreEngine;
```

Factory **stateless**. Sin overloads. Sin `async`. Sin `Promise`.

### 5.6 Conceptos Freeze (nombres inmutables)

`RestoreRequest` · `RestoreResult` · `RestoreStatus` · `RestoreStatistics` · `RestoreError` · `SessionRestoreEngine` · `RestoreValidator` · `RestoreReport`

### 5.7 RestoreErrors — nombres Freeze + semántica D67

| Nombre Freeze | Semántica en D67 |
|---------------|------------------|
| `MissingSnapshot` | **Missing restore payload** (records ausentes / vacíos). **No** implica D69 `SessionSnapshot`. |
| `CorruptedSnapshot` | Record corrupto / ilegible |
| `UnsupportedVersion` | Forma de payload no soportada (sin campo `schemaVersion` en Record D66) |
| `RegistryFailure` | `register` rechazado (p. ej. id duplicado) |
| `ValidationFailure` | Validator rechazó estructura / ids / required / metadata / duplicados |

### 5.8 RestoreValidator — alcance Freeze

**Permitido únicamente:**

- Estructura del record
- Ids
- Required fields
- Metadata (objeto plano serializable si presente)
- Duplicados (dentro del batch)

**Prohibido:**

- `schemaVersion` (rompe Freeze D66)
- Mutación de records
- Escritura a Registry
- I/O

### 5.9 RestoreReport — data-only Freeze

`warnings` · `errors` · `restoredItems` · `skippedItems` · `elapsedTime`

### 5.10 Barrel público (`restore/index.ts`)

**Allowlist prevista:** tipos Freeze + `createSessionRestoreEngine` + exports públicos de validator/report según implementación D67.1–D67.8.

**Prohibido:**

- Re-export desde `session/index.ts`
- Constantes IndexedDB
- APIs de autosave / snapshot product / workspace registries

### 5.11 Compatibilidad D65 / D66 (intacta)

| Superficie | Acción D67 |
|------------|------------|
| `SessionDefinition` / `SessionState` / `SessionRegistry` | **Sin modificación** |
| `SessionPersistenceRecord` / Adapter / Bridge / Serializer / Deserializer | **Sin modificación** |
| Adapter API | Sigue `save` / `load` / `clear` — **no ampliar** |
| `session/index.ts` | **Sin** re-export de `restore/*` |
| `persistence/index.ts` | **Sin** cambios requeridos por D67.0 |

## 6. Hard Rules

| ID | Regla |
|----|-------|
| **HR-no-react** | `restore/*` no importa ni usa React. |
| **HR-no-hooks** | Sin Hooks. |
| **HR-no-context** | Sin Context. |
| **HR-no-indexeddb** | Restore no accede IndexedDB; solo recibe records (Adapter = sole I/O D66). |
| **HR-no-localstorage** | Sin LocalStorage. |
| **HR-no-ui** | Sin UI / botones / auto-restore. |
| **HR-no-components** | Restore no crea componentes — solo datos. |
| **HR-register-public** | Única escritura: `SessionRegistry.register(...)`. Nunca Map interno / `entries.set`. |
| **HR-deserialize-reuse** | Reutilizar exclusivamente `deserializeRegistry` / `deserializeSession` (D66). Sin segundo mecanismo. |
| **HR-no-schema-version** | Validator / tipos restore **no** introducen `schemaVersion` sobre el Record D66. |
| **HR-engine-sync** | `restore(request): RestoreResult` — sync; sin Promise/overloads. |
| **HR-engine-stateless** | Engine sin estado; `registry` solo vía `RestoreRequest`. |
| **HR-Restore-Imports** | Ver §6.1 (validator-enforced). |
| **HR-barrel-isolation** | `session/index.ts` **must NOT** export `restore/*`. |
| **HR-d65-d66-compat** | Compatibilidad total con D65/D66; sin refactor de Persistence ni Registry. |

### 6.1 HR-Restore-Imports (congelada)

`restore/*` **may import only:**

- Session types / Definition / State / `SessionId` / `SessionEntry`
- `SessionRegistry` (tipo + uso de API pública)
- Persistence **deserialize** helpers (`deserializeRegistry`, `deserializeSession`, tipo `SessionPersistenceRecord`)
- Shared utils no-UI (si aplica)

`restore/*` **must never import:**

- Window · Tabs · Content · Series · Workspace · Dock · Layout · Toolbar · Inspector · UI · React

## 11. Criterios oficiales de cierre

D67 se considera **COMPLETE** cuando **todas** las condiciones siguientes son verdaderas:

1. Existe el módulo `src/components/session/restore/` con la estructura Freeze.

3. Validator sin `schemaVersion`; checks Freeze únicamente.
4. Engine reutiliza `deserializeRegistry` (D66) — sin segundo mecanismo.
5. Única escritura: `SessionRegistry.register`.
6. Solo SessionRegistry reconstruido; workspace registries deferred documentado.
7. Barrel D65 intacto; restore no se re-exporta desde `session/index.ts`.
8. **HR-Restore-Imports** PASS.
9. `tsc --noEmit` PASS.
10. `validate:d67-gate` PASS.
11. `validate:d65-gate` + `validate:d66-gate` PASS (compatibilidad).
