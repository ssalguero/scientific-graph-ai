## 2. Objetivo de la serie

Entregar la capa híbrida congelada en D63.0:

- Content Types opacos (`ContentDefinition` / `ContentHostProps`)
- ContentRegistry SSOT (independiente de TabRegistry)
- ContentSlots tipados
- TabSeriesBridge 1↔1 mapping-only
- ContentBridge resolve sin caché
- ContentHost presentacional sin ownership
- Barrel `content/index.ts`
- ContentIntegration library-only (never mounted by `page.tsx`)
- Validators + umbrella gate

**Sin** renderers científicos, **sin** WindowManager product lifecycle, **sin** acoplamiento a Workspace shell, **sin** cambios visuales.

## 3. Arquitectura liberada

```text
Window
  │
  ▼
WindowTabsBridge              (D61 · LOCKED)
  │
  ▼
TabRegistry / TabSelection    (D61–D62 · LOCKED)
  │
  ▼
TabDocumentSwitch             (D62 · LOCKED)
  │
  ▼
OpaqueContentHandle           (D62 · LOCKED · unchanged)
  │
  ▼
ContentBridge.resolve         (D63 · sole resolution path)
  │
  ▼
ContentRegistry.get(id)       (D63 · SSOT · ≠ TabRegistry)
  │
  ▼
ContentDefinition             (D63 · { id, kind, title })
  │
  ▼
ContentHost                   (D63 · props-only · null/children)

Parallel (official axis · 1↔1):
  TabId ↔ SeriesId via TabSeriesBridge   (D63 · mapping / queries only)
```

| ContentRegistry = único SSOT · ≠ TabRegistry | **CONFIRMED** |
| ContentBridge sin caché (`Registry.get` cada resolve) | **CONFIRMED** |
| ContentHost sin ownership | **CONFIRMED** |
| TabSeriesBridge 1↔1 · sin Selection/Activation/Focus | **CONFIRMED** |
| Barrel `content/` · sin leak `windows/index.ts` | **CONFIRMED** |
| Library-only integration · never mounted by `page.tsx` | **CONFIRMED** |
