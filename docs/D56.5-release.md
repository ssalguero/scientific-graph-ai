---

## 2. Objetivo de D56

Construir la infraestructura para Floating Windows de modo que cualquier panel desacoplable pueda existir como ventana flotante administrada por el Window Manager existente — **sin** resize, snapping, docking avanzado, tabs, persistencia ni multi-monitor.

---

## 3. Arquitectura final implementada

```text
WindowManager (D55 SSOT — API unchanged)
        │
Home / page.tsx root
        │
WorkspaceLayout
        │
WorkspacePanels
        ├── DockRoot
        ├── FloatingWindowBridge   ← único useWindowContext()
        │         │
        │         └── FloatingWindowLayer windows={[]}
        │                   │
        │                   └── FloatingWindow (presentational)
        └── Toasts
```

| Capa | Rol | Hooks |
|------|-----|-------|
| `WindowManager` | SSOT / Provider D55 | owns state |
| `FloatingWindowBridge` | Adapta contexto → props | `useWindowContext` only |
| `FloatingWindowLayer` | Render colección visible | none |
| `FloatingWindow` | Chrome presentacional | none |

Lifecycle de render (documental): show/hide ↔ `visible` / `minimize` / `restore` — **sin** nuevos métodos en WindowAPI.

## 5. API Freeze respetada

### D56 (congelada)

- `FloatingWindowModel` — `id`, `title`, `x`, `y`, `width`, `height`, `zIndex`, `visible`, `content?`
- `FloatingWindowProps` — `{ window: FloatingWindowModel }`
- `FloatingWindowLayerProps` — `{ windows: readonly FloatingWindowModel[] }`

Sin callbacks, handlers, drag, resize, snap, dock, tabs, persistence, monitor, minimized/maximized/focused/active.

### D55 (intacta)

`WindowAPI` permanece: `create`, `register`, `unregister`, `activate`, `focus`, `minimize`, `restore`, `close`, `get`, `getAll`.  
**No** se añadieron `show` ni `hide`.
