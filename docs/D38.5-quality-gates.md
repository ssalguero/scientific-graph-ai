## 6. Modelo general de Quality Gates

Cada gate define: Objetivo · Owner · Momento · Evidencia · PASS · FAIL · Acción ante FAIL.

Momentos: Discovery → BUILD → Testing → Certification → Release.

**Versión de catálogo:** QG-PROD3 **v1.0** (incremento solo vía Amendment aprobado).

---

## 7. Architecture Gate

| Campo | Contenido |
|-------|-----------|
| **Objetivo** | Verificar ownership, boundaries, capas y cumplimiento del Architecture Freeze |
| **Owner** | Arquitectura / gobernanza documental |
| **Momento** | Discovery · Certification |
| **Evidencia** | Diff vs D38.2 Ownership · ausencia de cambios en barrels GRAPH · acta épica |
| **PASS** | Ownership D38.2 §11 respetado · capas domain/interaction/rendering intactas · scope no ampliado sin Amend |
| **FAIL** | Cambio a barrels D29–D35 · violación Out of Scope · ownership incumplido |
| **Acción ante FAIL** | Stop BUILD · remediation o Amendment D38.x |

---

## 8. API Freeze Gate

| Campo | Contenido |
|-------|-----------|
| **Objetivo** | Verificar API Freeze D25–D35 · VGB-R1 · schema · exports · barrels |
| **Owner** | Mantenedor GRAPH / prod2e |
| **Momento** | BUILD · Testing · Certification |
| **Evidencia** | Registry API Freeze · ausencia schema bump · VGB-R1 (efímeros no persistidos) |
| **PASS** | schemaVersion sin bump · barrels sin exports nuevos no autorizados · VGB-R1 OK |
| **FAIL** | schema bump · breach VGB-R1 · exports no autorizados en barrels congelados |
| **Acción ante FAIL** | Stop · Amendment API si corresponde · no merge |

---

## 9. Documentation Gate

| Campo | Contenido |
|-------|-----------|
| **Objetivo** | SSOT vigente · consistencia · trazabilidad · Amendments · Resolution Notes |
| **Owner** | Gobernanza documental |
| **Momento** | Discovery · Certification · Release |
| **Evidencia** | Docs de épica · Decision Log · referencias a D38.2–D38.5 |
| **PASS** | SSOT citado · sin contradicción con D38.2/D38.4 · Amendments registrados si hubo desviación |
| **FAIL** | Docs contradictorias al SSOT · edición silenciosa de Freeze · falta de trazabilidad |
| **Acción ante FAIL** | Corregir docs · Amendment/RN · re-certificar |

---

## 10. Performance Gate

| Campo | Contenido |
|-------|-----------|
| **Objetivo** | Criterios de diseño para export PNG · SVG · memoria · tiempos (sin mediciones reales en D38) |
| **Owner** | Épica BUILD (EXPORT-1+) |
| **Momento** | Testing · Certification |
| **Evidencia** | Informe de medición en BUILD (a definir umbrales en D39 Discovery; **no fijados aquí como hechos**) |
| **PASS** | Mediciones documentadas según umbrales acordados en Discovery de la épica · sin regresión grave reportada |
| **FAIL** | Umbrales de épica incumplidos · ausencia de evidencia cuando el Release la exige |
| **Acción ante FAIL** | Optimizar · documentar waiver vía Resolution Note / Amendment si aceptable |

---

## 11. Build Gate

| Campo | Contenido |
|-------|-----------|
| **Objetivo** | `tsc` · integridad del proyecto · compatibilidad |
| **Owner** | Épica BUILD |
| **Momento** | BUILD · Testing · Certification |
| **Evidencia** | `npx tsc --noEmit` PASS · build/app íntegra (comandos existentes; **sin nuevos scripts en D38**) |
| **PASS** | TypeScript limpio · proyecto compilable/integro |
| **FAIL** | Errores tsc · rotura de integridad |
| **Acción ante FAIL** | Fix antes de continuar |

---

## 12. Regression Gate

| Campo | Contenido |
|-------|-----------|
| **Objetivo** | Obligatoriedad `validate:prod2e-gate` · regresión funcional · evidencia |
| **Owner** | Mantenedor GRAPH / prod2e-gate |
| **Momento** | Testing · Certification · Release |
| **Evidencia** | Log `validate:prod2e-gate` PASS · smokes de épica cuando existan |
| **PASS** | prod2e-gate PASS · sin regresión VGB/GRAPH reportada |
| **FAIL** | prod2e-gate FAIL · regresión cross-type |
| **Acción ante FAIL** | Stop Release · fix · re-run |

---

## 13. Governance Gate

| Campo | Contenido |
|-------|-----------|
| **Objetivo** | Scope Freeze · Governance · Roadmap · Architecture Freeze |
| **Owner** | Gobernanza documental |
| **Momento** | Discovery · Certification · Release |
| **Evidencia** | Checklist vs D38.2–D38.4 · Decision Log actualizado si aplica |
| **PASS** | Scope Freeze respetado · Roadmap Official seguido · sin edición silenciosa |
| **FAIL** | Violación de Freeze/Governance/Roadmap sin Amendment |
| **Acción ante FAIL** | Stop · Amendment o rollback de alcance |

---

## 14. Release Gate

| Campo | Contenido |
|-------|-----------|
| **Objetivo** | Requisitos mínimos para declarar una épica **READY** |
| **Owner** | Lead de épica + checklist D38 |
| **Momento** | Release |
| **Evidencia** | PASS de gates previos según Dependency Matrix · acta · docs |
| **PASS** | Todos los gates dependientes PASS · handoff documentado · CA de épica PASS |
| **FAIL** | Gate previo FAIL · evidencia incompleta |
| **Acción ante FAIL** | No declarar READY · completar gates |

## 17. Quality Gate Change Policy

| Cambio | Instrumento |
|--------|-------------|
| Cambio de criterios | Amendment D38.x |
| Nuevo Gate | Amendment o nueva fase oficial |
| Cambio editorial | Resolution Note |
| Modificación silenciosa | **Prohibida** |

## 18. Gate Versioning Policy

| Cambio | Acción |
|--------|--------|
| Cambio de criterio PASS/FAIL | Amendment D38.x |
| Nuevo gate | Nueva versión del catálogo |
| Eliminación de gate | Nueva fase de Freeze |
| Cambio editorial | Resolution Note |

**Regla:** la versión del catálogo de Quality Gates se incrementa **únicamente** mediante Amendment aprobado.
