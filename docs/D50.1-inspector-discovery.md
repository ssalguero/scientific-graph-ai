## 5. Architecture Freeze

### 5.1 Árbol objetivo (congelado)

```text
WorkspaceLayout
├── Sidebar
├── WorkspaceContent
│     ├── Toolbar
│     └── Canvas
└── WorkspacePanels
      ├── Inspector          ← NUEVO (D50) — dock vacío
      └── Existing overlays  ← sin cambios (toasts, etc.)
```

### 5.2 Principios

| Principio | Contrato |
|-----------|----------|
| Presentational only | Slots `ReactNode`; sin handlers propios |
| Empty first | Sin tools; children opcionales / vacíos |
| Zero visual impact | `visible={false}` en el wiring inicial |
| Ownership | Page compone children/overlays; Inspector no conoce dominio |
| Dependency direction | `UI_TOKENS` → Inspector → Page → Workspace — **nunca** Inspector → page / scientific / graph / analysis |
| Move-only | Sin rediseño; sin nuevo comportamiento; foundation = shell vacío |

```text
ARCHITECTURE = FROZEN
HOST = WorkspacePanels
SHELL = EMPTY DOCK
ANALYSIS INSPECTOR = NOT PART OF THIS ARCHITECTURE
```

```ts

  visible: boolean

}

  children: ReactNode
}

}
```

### 6.2 Slot API (composición)

```tsx

  {children}
</Inspector>

  {children}
```

`InspectorPanel` envuelve el cuerpo del dock (slot puro).  
`InspectorLayout.tsx` forma parte del **File Freeze** (shell interno de layout); la API pública congelada de visibilidad/ancho vive en **`InspectorProps`**.

```ts

selectionType
reserved
```

**Indicar explícitamente:** no provider, no hooks, no lógica de selección en D50. Solo tipos/interfaces reservados en `types.ts` cuando se implemente en D50.2.

```text
API = FROZEN
InspectorProps · InspectorPanelProps · InspectorSectionProps
InspectorContext = RESERVED (no logic)
```
