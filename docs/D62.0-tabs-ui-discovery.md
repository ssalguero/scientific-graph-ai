## 1. Executive Summary

**Tabs UI / Document Switch / Selection Policy Foundation** es la capa **operable** sobre Window Tabs Foundation (D61): policy de selección, document switch React-agnóstico y chrome presentacional — **sin** wiring Tab↔Series, **sin** persistencia, **sin** extensión WindowAPI.

D62 crea la operabilidad (mutator aditivo, selection policy, document switch, `tab-ui/`, barrels, validators). D62.0 **solo** congela arquitectura; no implementa.

| Principio | Contrato D62 |
|-----------|--------------|
| Operabilidad sobre D61 | Policy + switch + UI antes de Tab↔Series (D63) |
| Mutator sin over-freeze | D62.0: existencia aditiva · D62.1: forma API |
| `activeTab` SSOT única | UI: `isActive = (tab.id === activeTab)` · sin dual-write `TabState` |
| Switch React-agnóstico | Opaque Content Handle en `tabs/` · React solo en `tab-ui/` |
| `tabs/**` puro | No React · No JSX · No TSX · No hooks · No Context · No DOM · No CSS |
| UI aislada | Namespace `tab-ui/` · barrel propio · sin leak `windows/index.ts` |
| Compatibilidad absoluta | WindowAPI / Series / Floating / Drag / Resize / Snap / Tabs D61 intactos |

```text
D62 = TABS OPERABILITY (policy + document switch + UI chrome)
D62 ≠ Tab↔Series wiring operativo (D63)
D62 ≠ persistencia / drag-reorder (D64+)
D62 ≠ extensión WindowAPI / campos tabs en Window*
D62 ≠ React dentro de tabs/**
D62 ≠ dual SSOT de activación (TabState vs activeTab)
D62 ≠ nombre Freeze del mutador Registry en D62.0
D62 ≠ dominio científico / datasets / charts
D62 ≠ product window population obligatoria
```

Equivalente incremental a:

```text
D55 → ventanas
D56 → floating
D57 → drag
D58 → resize
D59 → snap
D60 → series
D61 → tabs (infra)
D62 → tabs UI / policy / switch
```

## 4. Exclusiones

D62 **no** implementa ni autoriza:

- Asociación operativa Tab ↔ Series / wiring `WindowSeriesBridge`
- Lifecycle WindowManager (open/close tab product flows)
- Campos tabs en `WindowDefinition` / `WindowState`
- Auto-selección / política “siguiente tab” **dentro** de `TabSelectionStore` o `TabRegistry.unregister`
- Dual-write / sync de activación vía companion `TabState` como SSOT paralelo
- Congelar en D62.0 el **nombre** del mutador Registry (`setState` / `update` / `replace` / etc.)
- Persistencia / drag-reorder de tabs
- Extensión de WindowAPI / SeriesRegistry / Layout Engine / Floating / Drag / Resize / Snap
- Dominio científico / datasets / gráficos / análisis
- React / JSX / ReactNode dentro de `tabs/**` (incl. Document Switch)
- **Cualquier código, validator o script en D62.0**
- Adelantar D62.1 o D63
