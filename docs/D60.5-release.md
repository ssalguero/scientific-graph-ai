## 2. Objetivo de la serie

Construir la infraestructura que permite que una ventana represente una Serie de forma desacoplada:

- Identidad (`SeriesId`, kinds, metadata)
- Registry (autoridad de series registradas)
- Selection (`selectedSeries` / `activeSeries` / `focusedSeries`)
- Window mapping (`WindowId ↔ SeriesId`) vía bridge externo

**Sin** datasets, gráficos, render, import/export, cálculos ni análisis.

---

## 3. Arquitectura implementada

```text
WindowId
   │
   ▼
WindowSeriesBridge     (mapa desacoplado — fuera de WindowRegistry / WindowAPI)
   │
   ▼
SeriesId
   │
   ├──► SeriesRegistry
   ├──► SeriesSelectionState  (+ SeriesSelectionBridge)
   └──► SeriesMetadata
```
