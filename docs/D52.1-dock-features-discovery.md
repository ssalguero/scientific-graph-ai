## 4. Arquitectura funcional

```text
DOCK_REGISTRY (seed, frozen)
            │
            │  hydrate
            ▼
      DockRegistry (live store)
            ▲
            │  mutate only via
DockRegistrationApi
            │
            ▼
       DockContext
      │            │
      │            │
Visibility API  Layout / Slots
      │            │
      └─────┬──────┘
            │
       Dock Features
       (+ DOCK_FEATURES flags)
```

### 4.1 Flujo obligatorio

```text
Registration API
       │
       ▼
   Registry
       │
       ▼
   Context
```

### 4.2 Separación de responsabilidades

| Capa | Responsabilidad | API |
|------|-----------------|-----|
| **DockRegistry** (live store) | Almacenar · consultar · enumerar | `get()` · `list()` · `has()` — **no modifica** |
| **DockRegistrationApi** | Ciclo de vida del registro | `register()` · `unregister()` — **único punto autorizado de mutación** |
| **DockContext** | Exponer modelo certificado | Lee Registry + APIs; sin store duplicado |
| **DOCK_REGISTRY** (seed) | Semilla D51 | `Object.freeze` · inspector-only · hidrata el live store |

**Contrato de acoplamiento:** Registration **nunca** es el store. Registry **nunca** expone `register` / `unregister`. Context **nunca** implementa un segundo registro.

## 5. Capacidades nuevas D52

### 5.1 Registration

Introducir `DockRegistrationApi`:

| Método | Rol |
|--------|-----|
| `register` | Añade entrada al Registry vivo |
| `unregister` | Quita entrada del Registry vivo |

Consulta / enumeración **exclusivamente** vía **DockRegistry**:

| Método | Rol |
|--------|-----|
| `get` | Lookup por id |
| `list` | Enumerar entradas |
| `has` | Existencia |

`DOCK_REGISTRY` permanece como **seed congelado** (compatibilidad D51).

### 5.2 Visibility

Nueva `DockVisibilityApi`.

Controla únicamente `DockState.activePanelIds`.

**No controla:** DOM · estilos · animaciones · renderizado de hosts.

### 5.3 Layout

Nueva `DockLayoutDefinition`.

Describe zonas. Inicialmente:

| Zona | Estado D52 |
|------|------------|
| `LEFT` | Definición |
| `RIGHT` | Definición |
| `BOTTOM` | Definición |
| `FLOATING` | Reserved (sin comportamiento) |
| `CENTER` | Reservado en `DockSide` D51; sin chrome D52 |

No implementa comportamiento de layout visual.

### 5.4 Slots

Nuevo `DockSlotDefinition`.

Asocia:

```text
Zone → Panel IDs
```

No modifica props ni comportamiento visual de `DockZone` / `DockPanel`.

### 5.5 Feature Flags

Nuevo `DOCK_FEATURES`.

| Contrato | Valor |
|----------|-------|
| Naturaleza | Flags internos |
| Default | Desactivados |
| Wiring UI | No cableados a UI |
| Efecto | Garantiza Zero UX |

```text
CAPABILITIES = registration · visibility · layout · slots · flags
INTERACTIONS = OUT OF SCOPE (D53+)
```

## 7. API Freeze

**Estado:** **API FROZEN** desde D52.1. Sin breaking changes durante D52. Toda modificación posterior requiere nueva fase.

### 7.1 Contratos nuevos (congelados)

Se congelan:

- `DockRegistration` / `DockRegistrationApi`
- `DockVisibility` / `DockVisibilityApi`
- `DockLayoutDefinition`
- `DockSlotDefinition`
- `DOCK_FEATURES`
- Superficie de consulta del **DockRegistry** vivo (`get` / `list` / `has`)

#### 7.1.1 DockRegistrationApi (verbatim preview)

```ts
type DockRegistrationApi = {
  register(entry: DockRegistryEntry): void;
  unregister(id: string): void;
};
```

#### 7.1.2 DockRegistry live query surface (verbatim preview)

```ts
type DockRegistryQuery = {
  get(id: string): DockRegistryEntry | undefined;
  list(): readonly DockRegistryEntry[];
  has(id: string): boolean;
};
```

#### 7.1.3 DockVisibilityApi (verbatim preview)

```ts
type DockVisibilityApi = {
  show(panelId: string): void;
  hide(panelId: string): void;
  isVisible(panelId: string): boolean;
  // Operates only on DockState.activePanelIds
};
```

#### 7.1.4 DockLayoutDefinition / DockSlotDefinition (verbatim preview)

```ts
type DockSlotDefinition = {
  side: DockSide;
  panelIds: readonly string[];
};

type DockLayoutDefinition = {
  slots: readonly DockSlotDefinition[];
  // Zones: left · right · bottom · floating (reserved)
};
```

#### 7.1.5 DOCK_FEATURES (verbatim preview)

```ts
const DOCK_FEATURES = {
  registration: false,
  visibility: false,
  layout: false,
  slots: false,
} as const;
```

## 9. Governance Freeze

**Estado:** **GOVERNANCE = FROZEN**.

| Regla | Significado |
|-------|-------------|
| `dock.features.noScientificImports` | Sin imports científicos / graph / analysis / page / supabase / stores en `docking/` |
| `dock.registry.singleSource` | Un único Registry vivo; sin estados duplicados de paneles |
| `dock.registry.noMutations` | Registry no expone `register` / `unregister`; mutaciones solo vía RegistrationApi |
| `dock.layout.definitionFrozen` | Layout definido únicamente mediante modelo (`DockLayoutDefinition` / slots) |
| `dock.visibility.noDomManipulation` | Visibilidad únicamente mediante estado (`activePanelIds`) |
| `dock.features.zeroUx` | Sin cambios visuales durante D52 |
| `dock.api.additiveOnly` | Todas las APIs evolucionan de forma aditiva |
| `dock.tokens.ssot` | Única fuente visual/numérica: `UI_TOKENS.dock` vía `DOCK_TOKENS` |
| `dock.state.modelOnly` | El estado Dock solo puede contener datos de modelo |

### 9.1 `dock.state.modelOnly` (detalle)

**Permitido en state / layout / slots / visibility:**

- IDs (`string`)
- Layout definitions
- Slot definitions
- Visibility (`activePanelIds`)
- Sizes numéricos (`DockSize`)

**Prohibido:**

- `HTMLElement`
- `ReactNode`
- `ref` / React refs
- `DOMRect`
- Cualquier referencia al DOM

Esto protege el futuro sistema de drag (D53) de acoplamiento prematuro al DOM.

### 9.2 Imports

| Prohibido en `docking/` | Permitido |
|-------------------------|-----------|
| `page.tsx` / `@/app/page` | React |
| Scientific / `@/lib/scientific` | `@/lib/ui/tokens` |
| Graph / Analysis | Módulos intra-`docking/` |
| supabase / stores / domain math | |

```text
GOVERNANCE = FROZEN
dock.features.* · dock.registry.* · dock.layout.* · dock.visibility.*
dock.api.additiveOnly · dock.tokens.ssot · dock.state.modelOnly
```

Al finalizar D52 deberá ser posible:

5. Controlar visibilidad únicamente a través del modelo (`DockVisibilityApi`)
