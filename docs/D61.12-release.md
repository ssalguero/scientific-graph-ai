## 2. Objetivo de la serie

Construir la infraestructura base para que una ventana contenga múltiples documentos/series mediante pestañas:

- Identidad (`TabId`, factory, validator)
- Tipos públicos (`TabDefinition` / `TabState` / `TabReference` / `TabEntry`)
- Registry SSOT (`TabRegistry` + Store)
- Selection (`activeTab` only)
- Bridges (`WindowTabsBridge`, `TabSelectionBridge`)
- Barrel único + validators

**Sin** UI, render, integración visual, auto-select ni wiring WindowManager.

---

## 3. Arquitectura entregada

```text
Window
   │
   ▼
WindowTabsBridge          ← solo adaptación WindowId → TabId[]
   │
   ▼
TabRegistry               ← SSOT TabId → Definition + State
   │
   ▼
TabSelectionStore         ← solo activeTab
   │
   ▼
Series                    ← contenido (D62+; no cableado en D61)
```

| Principio | Estado |
|-----------|--------|
| Autoridad única por concern | **CONFIRMED** |
| Factories puras | **CONFIRMED** |
| TabRegistry no conoce Window/Series | **CONFIRMED** |
| Definition ⊥ State (TabEntry companion) | **CONFIRMED** |
| Sin campos tabs en WindowDefinition / WindowState | **CONFIRMED** |
| Tabs exportable solo desde `tabs/index.ts` | **CONFIRMED** |
| Sin UI / React / Context | **CONFIRMED** |
| Sin auto-select / next-tab policy | **CONFIRMED** |
