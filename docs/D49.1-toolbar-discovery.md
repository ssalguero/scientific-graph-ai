## 3. Baseline — Toolbar actual (LOCKED)

### 3.1 Ubicación

| Rol | Path |
|-----|------|
| Toolbar JSX | `src/app/page.tsx` (~L19722–19787) |
| Slot host | `WorkspaceContent` (`toolbar?: ReactNode`) |
| Shell | `WorkspaceLayout` → `WorkspaceContent` → `toolbar` + `workspace` |

El chrome **no** es un módulo propio hoy. Se pasa como Fragment al slot congelado D47:

```text
WorkspaceLayout
└── WorkspaceContent
      ├── toolbar   ← Fragment inline en page.tsx (TARGET D49)
      └── workspace ← secciones científicas (fuera de D49)
```

### 3.2 Estructura vertical (orden congelado)

```text
header
  ↓
nav (WorkspaceTab × N)
  ↓
WorkflowSessionIndicator (opcional)
  ↓
session paragraph (condicional)
  ↓
LabUsageProfileSelector
```

| # | Elemento | Notas |
|---|----------|-------|
| 1 | `<header>` | Marca “Scientific Graph AI” + subtítulo condicional (home / welcome hint) |
| 2 | `<nav role="tablist">` | Tabs vía `WorkspaceTab` local en `page.tsx` (~L515–559) |
| 3 | `WorkflowSessionIndicator` | Condicional (`showWorkflowSessionIndicator` + plan activo) |
| 4 | `<p>` session context | Si sección ≠ `home` |
| 5 | `LabUsageProfileSelector` | Controlled: `value` / `onChange` / `persistenceBadgeClassName` |

### 3.3 Handlers / ownership (permanecen en page)

| Control | Handler / dato | Owner |
|---------|----------------|-------|
| Tabs `onSelect` | `selectWorkspaceSection` | `page.tsx` |
| Tab list | `visibleWorkspaceTabs` | `page.tsx` |
| Analysis badge | `activeVisibilityToggleCount` | `page.tsx` |
| Workflow `onCancel` | `cancelGuidedWorkflow` | `page.tsx` |
| Profile `onChange` | `setLabUsageProfile` | `page.tsx` |
| Profile `value` | `labUsageProfile` | `page.tsx` |

**Decisión congelada:** AdaptiveToolbar **nunca** importa `WorkspaceTab`, `WorkflowSessionIndicator`, ni `LabUsageProfileSelector`. Siguen siendo children compuestos por `page.tsx`.

### 3.4 Clases / notas visuales (inventario; no migrar en D49.1)

| Elemento | Clases / tokens observados |
|----------|----------------------------|
| Title | `text-xl sm:text-2xl font-bold text-[var(--app-heading)] tracking-tight` |
| Subtitle / session | `text-[var(--app-text-muted)]` (+ size variants) |
| Nav | `flex flex-wrap gap-1.5 border-b border-[var(--app-border)] pb-2` |
| Tab active | accent border/bg + `text-white` + `shadow-sm` |
| Tab inactive | `btnOutline` + size classes |
| Badge active | `bg-white/20` |
| Host spacing | `WORKSPACE_TOKENS.inner` incluye `space-y-3` entre siblings del main column |

Hardcoded no-var observados (documentados; no rediseñar en D49): `text-white`, `bg-white/20`.

## 4. Decisiones congeladas

| Decisión | Valor |
|----------|--------|
| Path del módulo | `src/components/toolbar/` (paralelo a `workspace/`; no bajo `ui/`) |
| Slot de montaje | `WorkspaceContent.toolbar` — **Workspace API intacta** |
| Wiring D49.3 | `<AdaptiveToolbar left={…} />`; `center` / `right` omitidos |
| `ToolbarGroup` / `ToolbarAction` | Infraestructura exportada; **no** participan del wiring D49.3 |
| `ToolbarOverflow` | Scaffold: `children` passthrough; sin responsive / MQ / observers / lógica |
| Move-only | **Same visual tree** + **same interactive tree** — **no** same HTML |
| Wrappers | `AdaptiveToolbar` + `ToolbarSection` permitidos |
| Ownership | Controles científicos / tabs / lab profile = `page.tsx` |
| Dependency direction | `UI_TOKENS` → Toolbar → Page → Workspace — **nunca** Toolbar → Workspace |

```text
MOVE-ONLY = SAME VISUAL TREE + SAME INTERACTIVE TREE
WRAPPERS ALLOWED
IDENTICAL HTML = NOT REQUIRED
OWNERSHIP = PAGE
OVERFLOW = SCAFFOLD ONLY
GROUP / ACTION = INFRASTRUCTURE ONLY (NO WIRING)
```

## 5. Move-only Contract

### 5.1 Se preservan

| Dimensión | Contrato |
|-----------|----------|
| Handlers | Mismos nombres y wiring desde `page.tsx` |
| Orden | Mismo orden de controles en el árbol interactivo |
| Textos | Mismos labels / copy |
| Clases finales | Mismas class strings en nodos de contenido (chrome de dominio vía tokens) |
| Spacing | Misma apariencia de espaciado (gap del stack vertical) |
| Apariencia | Pixel-perfect / sin rediseño |
| Comportamiento | Mismos condicionales y resultados de interacción |

### 5.2 Se permite

- Wrappers propios del dominio Toolbar (`AdaptiveToolbar`, `ToolbarSection`)
- Bridge `TOOLBAR_TOKENS` → `UI_TOKENS.toolbar.*` para clases del shell del dominio

### 5.3 No se exige

- Árbol HTML idéntico al baseline pre-D49
- Ausencia de nodos de dominio Toolbar en el DOM

### 5.4 Ownership

```text
page.tsx owns:
  WorkspaceTab
  WorkflowSessionIndicator
  LabUsageProfileSelector
  all handlers / conditionals / badges

AdaptiveToolbar owns:
  layout slots (left / center? / right?)
  presentational section wrappers
  TOOLBAR_TOKENS chrome only
```

AdaptiveToolbar **no** importa componentes científicos ni workflow/lab chrome.

## 7. Tokens — bridge freeze

### 7.1 Dirección

```text
UI_TOKENS.toolbar.*
        ↓
  TOOLBAR_TOKENS
```

Bridge **100%**. Valores literales de clase viven solo en Design Tokens v2 (`src/lib/ui/tokens.ts`). Materialización aditiva de `UI_TOKENS.toolbar` = **D49.2** (no D49.1).

### 7.2 `ToolbarTokens.ts` (contrato documental)

```ts
// Documental — NO implementado en D49.1
import { UI_TOKENS } from "@/lib/ui/tokens";

export const TOOLBAR_TOKENS = {
  root: UI_TOKENS.toolbar.root,
  // section / group / action / overflow → solo referencias
} as const;
```

**Prohibido en `ToolbarTokens.ts`:**

- String literals
- Classnames inline
- CSS
- Hex / paletas ad-hoc

**Solo referencias** a `UI_TOKENS`.

### 7.3 Keys previstas (documental; forma exacta en D49.2)

Espaciado, height, gap, padding, border, background, radius, shadow — compuestos desde primitivos / semantic tokens existentes. Sin nuevos hex. Sin romper `WORKSPACE_TOKENS` / Sidebar / theme public APIs.

## 8. Governance preview (D49.4 — no implementar en D49.1)

Familia congelada `toolbar.*` (case ids previstos para validators):

| Case id | Objetivo |
|---------|----------|
| `toolbar.noTailwind` | Sin class strings ad-hoc en componentes toolbar; clases solo vía `TOOLBAR_TOKENS` |
| `toolbar.noHex` | Sin `#…` / slate / zinc ad-hoc en el módulo toolbar |
| `toolbar.tokensOnly` | Estilos del dominio solo a través de `TOOLBAR_TOKENS` → `UI_TOKENS` |
| `toolbar.noScientificImports` | Sin imports graph / analysis / scientific / supabase / stores |
| `toolbar.noWorkspaceImports` | Sin imports de `@/components/workspace` (prohibida dependencia inversa) |
| `toolbar.noState` | Sin hooks / `useState` / estado en el módulo toolbar |
| `toolbar.noBusinessLogic` | Sin cálculos, decisiones ni handlers propios |
| `toolbar.moveOnly` | Mismo árbol visual + interactivo; ownership en page; wrappers de dominio OK |
| `toolbar.apiFreeze` | Interfaces públicas coinciden con el freeze D49.1 |
| `toolbar.barrelStable` | `index.ts` exporta exactamente el set público congelado |

**Ownership ban adicional (move-only validator):** el módulo `toolbar/` no debe importar `WorkspaceTab`, `WorkflowSessionIndicator`, ni `LabUsageProfileSelector`.

**D49.1 no crea scripts ni package.json entries.**
