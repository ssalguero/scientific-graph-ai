## 1. Executive Summary

**Series Alignment Foundation** es la infraestructura que permitirá que una ventana represente una **Serie** de datos de forma **completamente desacoplada** del dominio científico — **sin** datasets, gráficos, render, import/export, cálculos ni análisis.

D60 crea **únicamente** la infraestructura (identidad, registry, selection state, metadata, bridges, tipos). No modifica comportamiento visible.

| Principio | Contrato D60 |
|-----------|--------------|
| Infraestructura primero | Tipos + registry + selection + mapping antes de lifecycle operativo |
| Desacoplamiento Window↔Series | Mapa externo vía `WindowSeriesBridge`; sin campos Series en Window* |
| Vocabulario del repo | `SeriesRegistry` (como WindowRegistry); `SeriesSelectionState` (como GeometryState); **sin** `*Store` |
| Subpaquete interno | Primer subdir de `windows/`: `series/` |
| Barrel único | Series exportable **solo** desde `windows/series/index.ts` |
| Zero UX / Zero React | No React · No JSX · No hooks · No Context · No DOM · No CSS |
| Compatibilidad absoluta | WindowAPI / Floating / Drag / Resize / Snap intactos |

```text
D60 = SERIES INFRASTRUCTURE (identity + registry + selection + window mapping)
D60 ≠ datasets
D60 ≠ charts / graphs / render
D60 ≠ scientific domain
D60 ≠ WindowAPI extension
D60 ≠ UI / Context / CSS
D60 ≠ product window population
```

### 5.1 Ubicación oficial (FROZEN)

### 5.2 Blueprint

```text
windows/
 ├── WindowAPI / WindowRegistry / WindowGeometryState   (D55–D58 · LOCKED)
 ├── Floating* / Drag* / Resize* / Snap*                 (D56–D59 · LOCKED)
 └── series/                                            (D60 · NEW subsystem)
      ├── Series Identity      (SeriesId, SeriesTypes, SeriesMetadata)
      ├── Series Registry      (SeriesRegistry + SeriesRegistryTypes)
      ├── Series Selection     (SeriesSelectionState + Types + Bridge)
      ├── WindowSeriesBridge   (WindowId ↔ SeriesId · decoupled)
      └── index.ts             (ÚNICO barrel de exportación Series)
```

### 5.3 Flujo conceptual (FROZEN)

```text
WindowId
   │
   ▼
WindowSeriesBridge     ← mapa desacoplado (fuera de WindowRegistry / WindowAPI)
   │
   ▼
SeriesId
   │
   ├──► SeriesRegistry
   ├──► SeriesSelectionState
   └──► SeriesMetadata
```

### 5.4 Principios arquitectónicos (FROZEN — sin fijar implementación)

| Principio | Significado |
|-----------|-------------|
| Autoridad única | Una autoridad por concern (Registry vs Selection vs Window mapping) |
| Factories puras | Creación sin side effects globales / singletons |
| Infraestructura desacoplada | Series no acopla Window* ni dominio científico |
| Sin UI | No React · No JSX · No hooks · No Context · No DOM · No CSS |
| Sin dependencias científicas | No imports de producto / análisis / datasets |

**No** se congelan estructuras internas concretas (p. ej. Map, clones, backends de storage) como requisito del Freeze. El Freeze congela **arquitectura y contratos**, no una implementación específica.

### 5.5 Continuidad D55–D59 (cita)

```text
D55 Registry → D56 Floating → D57 Drag → D58 Resize → D59 Snap → D60 Series Alignment
```

Autoridad real existente (sin redefinir): `WindowRegistry`, `WindowGeometryState`, bridges. D60 **reutiliza ese vocabulario** (`Registry`, `*State`, Bridge) y **no** introduce `*Store`.

### 7.2 Contratos Series a congelar (superficie)

| Símbolo | Rol |
|---------|-----|

| Bind / unbind / lookup Window↔Series | `WindowSeriesBridge` | Fuera de WindowRegistry |

## 8. Hard Rules (FROZEN)

| ID | Regla |
|----|-------|
| `HR-no-series-on-window-definition` | **Prohibido** agregar campos Series a `WindowDefinition` |
| `HR-no-series-on-window-state` | **Prohibido** agregar campos Series a `WindowState` |
| `HR-series-barrel-only` | Series exportable **solo** desde `src/components/windows/series/index.ts` |
| `HR-no-windows-barrel-leak` | **Prohibido** `export … from "./series"` (o cualquier Series*) en `windows/index.ts` |
| `HR-no-touch-window-surfaces` | No modificar WindowAPI, WindowRegistry, WindowGeometryState, Floating*, Drag*, Resize*, Snap* |
| `HR-zero-ui` | Sin cambios de UI / visuales / comportamiento existente |
| `HR-no-store-naming` | No introducir nombres `*Store` en `series/**` |

Estas Hard Rules son **irreversibles** a efectos de D60–D64: acoplar Series a `WindowDefinition`/`WindowState` o filtrar Series por el barrel público de `windows/` crea deuda estructural.
