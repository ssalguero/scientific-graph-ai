## 1. Objetivo

Documentar la infraestructura actual del shell para permitir una migración **move-only** hacia un sistema de docking desacoplado.

Congelar:

- Baseline del shell actual (árbol, ownership, render order, z-index)
- Inventario de paneles / drawers / overlays / zonas futuras
- Future Dock Tree
- Scope / out-of-scope de D51
- Docking API Freeze (tipos + `DOCK_PANEL_IDS`)
- File Freeze del módulo `docking/`
- Tokens Bridge (`DOCK_TOKENS = UI_TOKENS.dock`)
- Registry Freeze (immutable, inspector-only)
- Context Freeze (read-only)
- Host Architecture (transparent passthrough)
- Wiring Freeze (documental only)
- Governance Freeze
- Validators Preview (12 checks para D51.4)
- Acceptance Criteria de esta microfase

**D51.1 no implementa código funcional.**

### 1.1 Principios

| Principio | Contrato |
|-----------|----------|
| Move-only | Envolver / hospedar; no rediseñar ni cambiar comportamiento |
| Zero UX changes | Con Inspector `visible={false}`, el árbol dock no aporta layout |
| API Freeze | Contratos congelados antes de D51.2; breaking changes = nueva fase |
| Append-only governance | STATUS y docs de release solo append; sin reescribir historia |

## 14. Host Architecture

**Estado:** **HOST ARCHITECTURE = FROZEN**.

| Component | Contract |
|-----------|----------|
| `DockRoot` | Provider only — `<DockProvider>{children}</DockProvider>` |
| `DockZone` | Fragment passthrough — `return <>{children}</>` |
| `DockPanel` | Fragment passthrough — `return <>{children}</>` |

### 14.1 Prohibitions

- No DOM wrappers (`div` / `section` / etc.)
- No layout
- No styling (`className`, tokens-as-classes on hosts)
- No behavior (drag, resize, visibility chrome)

### 14.2 Zero UX invariant

If `Inspector` returns `null` (`!visible`), the dock subtree contributes **nothing** to the DOM. That preserves the D50 empty-dock objective.

```text
HOST ARCHITECTURE = FROZEN
TRANSPARENT HOSTS = REQUIRED
NO DOM WRAPPERS
NO LAYOUT
NO STYLING
NO BEHAVIOR
```
