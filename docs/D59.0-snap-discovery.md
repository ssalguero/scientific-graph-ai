## 1. Executive Summary

**Snap Foundation** es la capa geométrica magnética que alinea edges de ventanas flotantes hacia targets (workspace, otras ventanas; Dock reservado D61) cuando la distancia ≤ umbral — **sin** alterar contratos públicos D55/D56 ni las triads internas D57/D58.

| Principio | Contrato D59 |
|-----------|--------------|
| Infraestructura primero | Motor + providers antes de wiring Drag/Resize |
| Move-first | Composition en drag (D59.2) antes que resize (D59.3) |
| Engine desacoplado | Solo `(geometry, targets, config) → geometry` |
| Edges-only | Solo `left \| right \| top \| bottom` |
| Providers extensibles | Terrain listo para `DockTargetProvider` en D61 |
| Zero UX visual | Sin guías, HUD, CSS de snap |
| Compatibilidad absoluta | WindowAPI / Floating APIs / triads Drag·Resize intactas |

```text
D59 = SNAP INFRASTRUCTURE (pure engine + target providers)
D59 ≠ visual guides / HUD
D59 ≠ dock snap / drop zones
D59 ≠ focus stacking / z-order policy
D59 ≠ WindowAPI extension
D59 ≠ product window population
```

---

## 2. Objetivo técnico

Incorporar un **WindowSnapEngine** puro y **SnapTargetProviders** tales que cualquier caller (Drag write-path, Resize write-path, tests, futuro Dock) pueda:

1. Colectar `SnapTarget[]` vía providers (fuera del Engine).
2. Invocar `applyMagneticSnap(geometry, targets, config)`.
3. Obtener nueva geometría alineada a edges dentro del umbral.
4. Escribir únicamente vía `WindowGeometryState` (callers D59.2+).

El Engine **nunca** conoce Drag, Resize, pointer, React, stores, providers ni WindowManager.

## 5. Arquitectura definitiva (FROZEN)

### 5.1 Blueprint

```text
page.tsx
 └── WindowManager
      ├── WindowState / WindowAPI                 (D55 FROZEN)
      ├── WindowGeometryState                     (SSOT)
      ├── SnapConfig                              (D59 · internal)
      ├── SnapTargetProviders                     (Workspace + Window; Dock = D61)
      ├── composeSnap (optional helper)           (NOT inside Engine file)
      ├── WindowSnapEngine                        (PURE · edges-only)
      ├── WindowDragBridge                        (caller · D59.2)
      ├── WindowResizeBridge                      (caller · D59.3)
      ├── Providers (Geometry / Drag / Resize)
      └── …
           └── FloatingWindowBridge → Layer → FloatingWindow
                (sin cambios de contrato; geometría ya snapeada)
```

### 5.2 Dependency direction (FROZEN)

```text
DragBridge ──► compose ──► Engine
ResizeBridge ──► compose ──► Engine
Providers ──► compose (emit targets)
Engine ──► ∅   (no deps hacia Drag / Resize / Manager / Providers / React / stores)
```

### 5.3 Composition write-path (FROZEN)

```text
candidateGeometry
     ↓
SnapTargetProvider[]  →  targets[]
     ↓
WindowSnapEngine.applyMagneticSnap(geometry, targets, config)
     ↓
snappedGeometry
     ↓
GeometryState.set   (solo callers; Engine no escribe stores)
```

```text
Drag (D59.2):   Math → compose(Snap) → GeometryState
Resize (D59.3): Math → GeometryConstraints → WorkspaceConstraints → compose(Snap) → GeometryState
```

Si el resultado de snap en un eje viola `GeometryConstraints` (min size), **descartar snap en ese eje**.

## 8. Contratos internos Snap (API Freeze D59 — FROZEN)

### 8.1 Edge Model (FROZEN)

Snap opera **únicamente** sobre los cuatro bordes derivados de `WindowGeometry`:

```ts
type WindowEdges = {
  left: number;    // = x
  right: number;   // = x + width
  top: number;     // = y
  bottom: number;  // = y + height
};
```

```text
OPERA SOBRE:   left | right | top | bottom
NUNCA SOBRE:   center | rotation | scale | transform | matrix
```

Storage permanece `{ x, y, width, height }`. Edges son vista derivada pura.

### 8.2 SnapConfig (FROZEN)

```ts
type SnapConfig = {
  enabled: boolean;           // master — default true
  threshold: number;          // seed default — e.g. 8
  axisThresholdX: number;     // autoridad eje X — default = threshold
  axisThresholdY: number;     // autoridad eje Y — default = threshold
};
```

**Autoridad de umbral en runtime:** `axisThresholdX` / `axisThresholdY`.  
`threshold` es el valor semilla al construir la config por defecto.

### 8.3 SnapTarget (FROZEN)

```ts
type SnapTargetKind = "workspace" | "window"; // "dock" reserved D61 — no emitir en D59

type SnapTarget = {
  axis: "x" | "y";
  position: number;
  edge: "left" | "right" | "top" | "bottom";
  kind: SnapTargetKind;
  sourceId?: string;
  priority: number;
};
```

### 8.4 SnapTargetProvider (FROZEN)

```ts
type SnapTargetContext = {
  candidateId: string;
  candidate: WindowGeometry;
  workspace: WorkspaceConstraints;
  geometries: ReadonlyMap<string, WindowGeometry>;
  config: SnapConfig;
};

type SnapTargetProvider = {
  readonly id: string;
  collect(ctx: SnapTargetContext): readonly SnapTarget[];
};
```

| Provider | Fase | Emite |
|----------|------|--------|
| `WorkspaceTargetProvider` | D59.1 | `kind: "workspace"` |
| `WindowTargetProvider` | D59.1 | `kind: "window"` |
| `DockTargetProvider` | **D61** | `kind: "dock"` (reservado) |

**Prohibido:** hardcodear sources de targets dentro de `WindowSnapEngine`.

### 8.5 WindowSnapEngine (FROZEN)

```ts
function applyMagneticSnap(
  geometry: WindowGeometry,
  targets: readonly SnapTarget[],
  config: SnapConfig
): WindowGeometry
```

```text
INPUT:   geometry + targets + config
OUTPUT:  geometry  (siempre objeto nuevo — geometry-immutable-input)

Engine NUNCA conoce:
  Drag · Resize · Pointer · Mouse · React · Stores ·
  Providers · WindowManager · DOM · sessions
```

Si `!config.enabled` → devolver nuevo objeto con la misma geometría (clone).

Umbral efectivo por eje:

```text
axis === "x" → config.axisThresholdX
axis === "y" → config.axisThresholdY
```

## 9. Reglas de prioridad (FROZEN)

Cuando dos targets compiten en el mismo eje dentro del umbral:

```text
Workspace  >  Window  >  Dock (reservado D61)
```

| kind | priority (menor = más prioritario) |
|------|-------------------------------------|
| `workspace` | `0` |
| `window` | `1` |
| `dock` (D61) | `2` |

---

## 10. Reglas de empate (FROZEN)

Resolución **por eje** (`x` y `y` independientes):

```text
1. nearest          → menor |edge - target.position|
2. if tie           → mayor prioridad (menor priority number)
                      Workspace > Window > Dock
3. if still tie     → stable lexicographic order:
                      kind asc → sourceId ("" si ausente) asc
                      → edge asc (left|right|top|bottom)
                      → position asc
```

Sin aleatoriedad. Sin “last writer wins”.

---

## 11. Reglas de determinismo (FROZEN)

```text
same (geometry, targets, config)
  ↓
same output geometry
ALWAYS
```

- Sin `Date`, `Math.random`, globales mutables, I/O, DOM.
- Mismo orden de evaluación de ejes y empates en todas las corridas.
- Validator prospectivo D59.4: `validate-d59-determinism` (N corridas, deep-equal).

## 12. Reglas de pureza del Engine (FROZEN)

`WindowSnapEngine` **prohíbe**:

| Prohibido | Motivo |
|-----------|--------|
| React / JSX | No UI |
| Hooks | No ciclo de vida React |
| Stores / GeometryState / WindowState | No lectura de SSOT |
| DOM / Pointer Events | No input |
| Providers (invocar o importar implementaciones) | Targets llegan como argumento |
| Refs | No mutables de instancia |
| Singleton / estado de módulo mutable | No identidad global |
| Closures que capturen estado externo mutable | Pureza matemática |
| Importar DragBridge / ResizeBridge / WindowManager | Desacoplamiento |

Permitido: funciones puras, clones inmutables, helpers `toEdges` / `fromEdges` locales.
