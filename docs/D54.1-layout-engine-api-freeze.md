## 1. Alcance del Freeze

Este freeze cubre **únicamente** la API pública del Layout Engine (shell espacial).

| Incluido en este freeze | Excluido (permanecen en freezes previos / otros módulos) |
|-------------------------|----------------------------------------------------------|
| `LayoutNode` | `DockState` v1 · Dock Interaction API (D53 → D60) |
| `LayoutTree` | `DockLayoutDefinition` / slots / tabs internos |
| `LayoutRegion` | Hosts Dock D51 (`DockRoot` / `DockZone` / `DockPanel`) |
| `LayoutVisibility` | Tokens CSS / `UI_TOKENS` |
| `LayoutConstraints` | Persistencia (D55) · Presets (D56) |
| `LayoutNodeType` | Consumers React (Sidebar / Inspector / Toolbar) |
| `LayoutNodeId` | |
| `LayoutEngine` | |
| `LayoutEngineProps` | |

Extensiones posteriores a D54 deben ser **aditivas** sobre estos contratos. Cambiar firmas o eliminar campos = **breaking** = **prohibido hasta D56 inclusive**, salvo approved architectural change.

## 3. Identificadores y tipos base

### 3.1 `LayoutNodeId`

| Símbolo | Rol congelado |
|---------|---------------|
| `LayoutNodeId` | Identificador estable de nodo (string brand o alias de `string`) |

IDs del default tree (previsto D54.2) deben ser **estables** (p. ej. `root`, `sidebar`, `workspace`, `toolbar`, `dock`, `inspector`, `floating-layers`, `overlays`).

### 3.2 `LayoutNodeType`

| Símbolo | Rol congelado |
|---------|---------------|
| `LayoutNodeType` | Discriminante del rol del nodo en el shell |

Valores conceptuales alineados al árbol de discovery:

```text
INSPECTOR
FLOATING_LAYERS
```

(La forma TypeScript exacta — union string o enum — es implementación D54.2; el **conjunto semántico** queda congelado.)

## 7. `LayoutNode`

Shape congelada:

| Campo | Tipo conceptual | Rol |
|-------|-----------------|-----|
| `id` | `LayoutNodeId` | Identidad |
| `type` | `LayoutNodeType` | Rol semántico |
| `parent` | `LayoutNodeId \| null` | Padre (`null` en Root) |
| `children` | `LayoutNodeId[]` | Hijos ordenados |
| `visibility` | `LayoutVisibility` | Visibilidad de modelo |
| `size` | tamaño de modelo (width/height según aplique) | Dimensiones declarativas |
| `constraints` | `LayoutConstraints` | Límites |
| `region` | `LayoutRegion` | Región espacial |

```text
LayoutNode = {
  id: LayoutNodeId
  type: LayoutNodeType
  parent: LayoutNodeId | null
  children: LayoutNodeId[]
  visibility: LayoutVisibility
  size: LayoutSize   // shape de tamaño de modelo; ver §7.1
  constraints: LayoutConstraints
  region: LayoutRegion
}
```

### 7.1 `size` (parte del nodo)

`size` es campo congelado de `LayoutNode`. La shape mínima de tamaño de modelo prevista:

```text
LayoutSize = {
  width?: number
  height?: number
}
```

Si la implementación introduce un alias `LayoutSize`, debe permanecer **compatible** con width/height opcionales numéricos. No es un símbolo de API Freeze de primer nivel en esta lista, pero el campo `LayoutNode.size` **sí** está congelado.

| Símbolo | Rol congelado |


## 9. `LayoutEngine`

| Símbolo | Rol congelado |
|---------|---------------|
| `LayoutEngine` | API pura de resolución y consulta del árbol |

### 9.1 Responsabilidades congeladas

| Operación | Contrato |
|-----------|----------|
| Resolve | `Tree → Tree normalizado` |
| Get node | Obtener `LayoutNode` por id |
| Get children | Obtener hijos de un nodo |
| Validate region | Validar `LayoutRegion` |
| Validate constraints | Validar `LayoutConstraints` |

### 9.2 Pureza

```text
NO REACT STATE
NO HOOKS
NO DOM
NO CSS / STYLES
NO SIDE EFFECTS
INPUT TREE → RESOLVER → NORMALIZED TREE
```

### 9.3 Forma de superficie (prevista)

La implementación D54.2 puede exponer `LayoutEngine` como:

- objeto/namespace con funciones, o
- clase/módulo con métodos estáticos/instancia sin estado React

**Congelado:** el conjunto de capacidades §9.1 y la pureza §9.2.  
**No congelado como firma única:** el estilo OO vs functional (elección D54.2), siempre que el barrel exporte el símbolo `LayoutEngine` y las operaciones existan.

| Símbolo | Rol congelado |
|---------|---------------|
| `LayoutEngineProps` | Props de entrada para construir/resolver el engine o su árbol |

Shape mínima congelada:

| Campo | Tipo conceptual | Rol |
|-------|-----------------|-----|
| `tree` | `LayoutTree` (opcional si hay default) | Árbol de entrada |

```text
LayoutEngineProps = {
```

Uso previsto en D54.3: `WorkspaceLayout` puede aceptar/usar props alineadas a este contrato al resolver el default tree. **No** implica Context React.
