## 1. Resumen ejecutivo

**D66 — Session Persistence Foundation** introduce la infraestructura desacoplada de persistencia local para el Session Registry, sin alterar el dominio Session liberado en D65.

Al cierre de D66 existirá un módulo independiente bajo `src/components/session/persistence/` con:

- contrato congelado de persistencia (`SessionPersistenceRecord`)
- Serializer / Deserializer puros con identidad round-trip
- puerto `SessionStorageAdapter` (único dueño de IndexedDB)
- `SessionPersistenceBridge` unidireccional (Registry → Storage)
- barrel público exclusivo `@/components/session/persistence`
- suite de validadores + certificación

**SessionRegistry permanece el SSOT.** Persistence solo observa y escribe; no restaura, no hace autosave inteligente, no versiona y no sincroniza en la nube.

D66 deja la tubería lista para que:

| Serie | Capacidad | Consume D66 sin refactor |
|-------|-----------|--------------------------|
| **D67** | Session Restore Foundation | `adapter.load()` + `deserialize*` + `registry.register` |
| **D68** | Autosave inteligente | `bridge.persist*` / mutación → persist |
| **D69** | Snapshots múltiples | superficie nueva; **no** reutiliza el Record Freeze de D66 como snapshot versionado |

## 2. Objetivos

### 2.1 Objetivo primario

Crear la **infraestructura desacoplada de persistencia** para Session Registry, certificada y lista para D67–D69 **sin refactorizar** el código de D66.

### 2.2 Objetivos específicos

1. Congelar la API de persistencia (`Record`, Adapter, Bridge, Serializer, Deserializer).
2. Aislar IndexedDB en un único Adapter (DB dedicada, store `sessions`).
3. Garantizar round-trip determinístico serialize ↔ deserialize.
4. Conectar de forma mínima y privada `SessionProvider` → Bridge (sin exponer Persistence al producto).
5. Certificar boundaries, import isolation y compatibilidad total con D65.

### 2.3 No-objetivos (esta serie)

Ver §8 Fuera de alcance.

## 3. Arquitectura

### 3.1 Ubicación del módulo

```text
src/components/session/persistence/
    SessionPersistenceTypes.ts
    SessionSerializer.ts
    SessionDeserializer.ts
    SessionStorageAdapter.ts
    SessionPersistenceBridge.ts
    index.ts
```

**Reglas de colocación:**

- **Nunca** dentro de `windows/`
- **Nunca** mezclado en el núcleo del Session Registry (archivos D65 top-level)
- Módulo **completamente separado** bajo `session/persistence/`
- **No** se re-exporta desde `src/components/session/index.ts`
- Entrada pública exclusiva: `@/components/session/persistence`

### 3.2 Flujo unidireccional

```text
SessionRegistry (SSOT)
        ↓
SessionPersistenceBridge
        ↓
SessionSerializer
        ↓
SessionStorageAdapter
        ↓
IndexedDB (ScientificGraphAI / store: sessions)
```

**Nunca al revés en D66.** Restore (lectura → deserialize → register) pertenece a D67 y **no** atraviesa el Bridge de escritura como camino de restauración automática.

### 3.3 Diagrama de dependencias

```text
                    ┌─────────────────────┐
                    │   SessionProvider   │
                    │ (owner privado via  │
                    │      useRef)        │
                    └──────────┬──────────┘
                               │ wires (private)
                               ▼
┌──────────────┐    ┌──────────────────────────┐
│ SessionRegistry│◄──│ SessionPersistenceBridge │
│    (SSOT)      │    │  persistSession          │
└──────────────┘    │  persistRegistry          │
                    └────────────┬─────────────┘
                                 │ uses
                    ┌────────────┼────────────┐
                    ▼                         ▼
         ┌──────────────────┐      ┌─────────────────────┐
         │ SessionSerializer│      │ SessionStorageAdapter│
         │ (pure functions) │      │ (sole IndexedDB I/O) │
         └──────────────────┘      └──────────┬──────────┘
                                              ▼
                                   IndexedDB ScientificGraphAI
                                   object store: sessions
```

`SessionDeserializer` existe en paralelo como infraestructura pura (sin side effects). D67 lo consumirá junto a `adapter.load()`.

### 3.4 Separación de responsabilidades

| Componente | Responsabilidad única | No hace |
|------------|----------------------|---------|
| **SessionPersistenceTypes** | Contratos Freeze (`Record`, `SessionStorageAdapter`, tipos Bridge) | I/O, React, lógica |
| **SessionSerializer** | `SessionEntry` → `SessionPersistenceRecord` (puro) | Registry, IndexedDB, React |
| **SessionDeserializer** | `SessionPersistenceRecord` → `SessionEntry` (puro) | `register`, crear sesiones, I/O |
| **SessionStorageAdapter** | `save` / `load` / `clear` sobre IndexedDB | Dominio, merge, Registry |
| **SessionPersistenceBridge** | Registry → serialize → `adapter.save` | Restore, autosave, conocer IDB |
| **persistence/index.ts** | Única exportación pública del módulo | Re-exportar constantes IDB |
| **SessionProvider** (D66.7) | Ownership privado Adapter+Bridge | Exponer Persistence en Context |

### 3.5 IndexedDB (contrato de almacenamiento — solo Adapter)

| Constante | Valor |
|-----------|-------|
| Database name | `ScientificGraphAI` |
| Object store | `sessions` |
| Key | `definition.id` |
| Valor | `SessionPersistenceRecord` (replace on put) |

**No** mezclar con la DB de proyectos locales (`scientific-graph-ai-local`). Constantes IDB viven **solo** dentro del módulo Adapter; el Bridge **nunca** las importa ni las conoce.

## 4. API Freeze

Toda superficie listada aquí queda **congelada** en D66.0. Implementación (D66.1+) debe respetarla exactamente. Ampliar el Adapter en D67 está **prohibido**.

### 4.1 SessionPersistenceRecord

```ts
interface SessionPersistenceRecord {
  definition: SessionDefinition;
  state: SessionState;
}
```

- Isomorphic a `SessionEntry` (D65).
- **No** metadata extra.
- **No** timestamps de persistencia.
- **No** cache.
- **No** versionado.
- **No** campos de snapshot.

### 4.2 SessionStorageAdapter

```ts
interface SessionStorageAdapter {
  save(
    record: SessionPersistenceRecord | SessionPersistenceRecord[]
  ): Promise<void>;

  load(): Promise<SessionPersistenceRecord[]>;

  clear(): Promise<void>;
}
```

Factory prevista: `createSessionStorageAdapter(): SessionStorageAdapter`.

| Método | Semántica Freeze |
|--------|------------------|
| `save` | Reemplaza el/los registro(s) por id (**put**). **Nunca merge.** Acepta uno o array. |
| `load` | Devuelve **todos** los records. Sin `load(id)`. |
| `clear` | Vacía el store. |

### 4.3 SessionPersistenceBridge

```ts
type SessionPersistenceBridge = {
  persistSession(id: SessionId): Promise<boolean>;
  persistRegistry(): Promise<void>;
};

createSessionPersistenceBridge(
  registry: SessionRegistry,
  adapter: SessionStorageAdapter
): SessionPersistenceBridge;
```

| Método | Flujo Freeze |
|--------|--------------|
| `persistSession(id)` | `registry.get(id)` → `serializeSession` → `adapter.save(record)` · `false` si no existe |
| `persistRegistry()` | `registry.list()` → orden de inserción estable → `serializeRegistry` → `adapter.save(records)` |

Bridge **agnóstico de IndexedDB**: solo conoce `SessionRegistry` + puerto `SessionStorageAdapter`.

### 4.4 Serializer API

```ts
serializeSession(entry: SessionEntry): SessionPersistenceRecord;
serializeRegistry(entries: readonly SessionEntry[]): SessionPersistenceRecord[];
```

- Funciones puras.
- Sin hooks, sin context, sin JSX, sin React.
- Sin I/O.

### 4.5 Deserializer API

```ts
deserializeSession(record: SessionPersistenceRecord): SessionEntry;
deserializeRegistry(
  records: readonly SessionPersistenceRecord[]
): SessionEntry[];
```

- Funciones puras.
- **Sin** side effects sobre Registry.
- **Sin** crear ni registrar sesiones.
- Solo preparan datos para que D67 restaure.

### 4.6 Barrel público (`persistence/index.ts`)

**Allowlist prevista:**

- Types: `SessionPersistenceRecord`, `SessionStorageAdapter`, `SessionPersistenceBridge` (y aliases tipados necesarios del Freeze)
- Runtime: `createSessionStorageAdapter`, `createSessionPersistenceBridge`, `serializeSession`, `serializeRegistry`, `deserializeSession`, `deserializeRegistry`

**Prohibido en barrel:**

- Constantes IDB (db name, store, keyPath)
- Re-export del barrel D65 (`session/index.ts` no re-exporta persistence)
- APIs de restore / autosave / snapshot

### 4.7 Compatibilidad D65 (API pública intacta)

El barrel `src/components/session/index.ts` **no cambia** su allowlist:

`SessionId` · `SessionMetadata` · `SessionEntry` · `createSessionRegistry` · `SessionContext` · `useSessionContext` · `SessionProvider` · `SessionBridge`

`SessionDefinition` y `SessionState` **no se modifican** (estructura ni IDs).

## 5. Hard Rules

| ID | Regla |
|----|-------|
| **HR-registry-ssot** | `SessionRegistry` es el único SSOT de sesiones en runtime. |
| **HR-registry-no-idb** | Registry **nunca** lee ni escribe IndexedDB. |
| **HR-adapter-sole-io** | Solo `SessionStorageAdapter` realiza I/O de persistencia Session. |
| **HR-adapter-api-freeze** | Adapter = exactamente `save` / `load` / `clear`. No ampliar en D67. |
| **HR-adapter-replace-no-merge** | `save` reemplaza registros; **nunca** merge. Dominio = Registry. |
| **HR-bridge-idb-agnostic** | Bridge no conoce DB names, stores, transactions ni keyPath. |
| **HR-bridge-unidirectional** | Bridge solo escribe (Registry → Storage). No restore automático. |
| **HR-serialize-pure** | Serializer = funciones puras; sin React / hooks / JSX. |
| **HR-deserialize-pure** | Deserializer = funciones puras; sin React / hooks / JSX. |
| **HR-serialize-identity** | `serialize(deserialize(x)) ≡ x` y `deserialize(serialize(x)) ≡ x` (igualdad estructural JSON). |
| **HR-persist-registry-fidelity** | `persistRegistry` = `list()` → orden estable → serialize → `save`. Sin sort, filter ni transform. |
| **HR-deserialize-no-register** | Deserializer no modifica Registry, no crea sesiones, no registra. Solo devuelve `SessionEntry[]`. |
| **HR-context-no-persistence** | `SessionContext` / `SessionAPI` **no** exponen Persistence, Bridge, Adapter ni `persist*`. |
| **HR-no-def-state-mutate** | Persistence no modifica `SessionDefinition` ni `SessionState` (Freeze D65). |
| **HR-no-extra-metadata** | No metadata / timestamps / cache / versionado en el Record. |
| **HR-no-autosave** | D66 no implementa autosave inteligente ni persist-on-mutate. |
| **HR-no-auto-restore** | D66 no restaura automáticamente al montar. |
| **HR-no-snapshots** | D66 no implementa snapshots múltiples (`SessionSnapshot` product = D69). |
| **HR-no-cloud-sync** | D66 no implementa sincronización en la nube. |
| **HR-persistence-import-isolation** | `persistence/*` no importa `windows/*`, `layout/*`, `toolbar/*`, `content/*`, `tabs/*`, `series/*`. |
| **HR-d65-compat** | Compatibilidad total con D65; barrel Session público intacto. |

## 6. Integración prevista

### 6.1 Owner: SessionProvider

En **D66.7** (no en D66.0):

- `SessionProvider` crea Adapter + Bridge en `useRef` (una vez por instancia).
- El Bridge se cablea al mismo `registry` que ya posee el Provider.
- Ownership **privado**: no se añade Persistence a `SessionContextValue` ni a `SessionAPI`.

### 6.2 Qué no se modifica

| Superficie | Acción D66 |
|------------|------------|
| `WindowManager` | **Sin cambios** |
| `page.tsx` | **Sin cambios** (sigue importando solo barrel `@/components/session`) |
| `SessionBridge` | **Sin cambios** (sigue siendo Window → `windowIds` sync only) |
| `session/index.ts` | **Sin re-export** de persistence |
| `SessionDefinition` / `SessionState` | **Freeze intacto** |

### 6.3 Comportamiento runtime en D66

- Wiring tipado presente (Provider posee Bridge).
- **Sin** llamadas automáticas a `persist*` en mutaciones (autosave = D68).
- **Sin** `load` + register al montar (restore = D67).
- Las APIs `persist*` / `load` / `deserialize*` existen para consumo futuro y certificación estructural.

### 6.4 Preparación para series siguientes

| Serie | Punto de extensión en D66 (sin refactor) |
|-------|------------------------------------------|
| D67 Restore | `adapter.load()` + `deserializeRegistry` + `registry.register` (ownership en capa Restore, no en Bridge write) |
| D68 Autosave | Llamadas a `bridge.persistSession` / `persistRegistry` desde el owner (Provider u orquestador), con política inteligente fuera de D66 |
| D69 Snapshots | Superficie nueva; no ensucia `SessionPersistenceRecord` |

8. `SessionDefinition` / `SessionState` sin cambios.
9. Import isolation PASS.
10. `tsc --noEmit` PASS.
