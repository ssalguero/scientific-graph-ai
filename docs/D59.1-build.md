## 4. Decisiones implementadas

| Decisión | Valor |
|----------|-------|
| Engine signature | `applyMagneticSnap(geometry, targets, config) → geometry` |
| Desacoplamiento | Engine no importa Drag / Resize / Manager / React / stores / providers |
| Edges-only | Solo `left \| right \| top \| bottom` vía `toEdges` |
| Config | `enabled` + `threshold` + `axisThresholdX` + `axisThresholdY` |
| Prioridad | `SNAP_PRIORITY`: workspace=0 · window=1 · dock=2 (reservado) |
| Empates | nearest → priority → kind → sourceId → edge → position |
| Determinismo | Sin Date/Random/globales; comparación total estable |
| Inmutabilidad | `cloneGeometry` / nuevos objetos; inputs no mutados |
| Providers | Solo contrato abstracto en Types; **sin** implementaciones runtime |
| SnapResult | No tipado — salida = `WindowGeometry` directa |

---

## 5. Algoritmo (certificado documental)

1. Si `!config.enabled` → `cloneGeometry(geometry)`.
2. Derivar `WindowEdges` del input (snapshot).
3. Por eje `x` / `y` (independientes): filtrar targets del eje; distancia `|edge - position|`; dentro de `axisThresholdX|Y`; elegir winner por reglas de empate.
4. Aplicar winner X (si hay) moviendo el edge indicado preservando width/height.
5. Aplicar winner Y (si hay) igual.
6. Devolver siempre un objeto nuevo.
