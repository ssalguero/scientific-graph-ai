## 2. Objetivo de D57

Habilitar posición dinámica de `FloatingWindow` mediante:

- Position Store (SSOT de `x`/`y`)
- WindowDragBridge (`beginDrag` / `updateDrag` / `endDrag`)
- Title bar Pointer Capture
- Bridge Mapping reactivo hacia `FloatingWindowModel[]`

Sin resize, snap, dock, constraints, persistencia ni transformaciones de coordenadas.

---

## 3. Arquitectura final certificada

```text
WindowManager
 ├── WindowState / WindowAPI          (D55 FROZEN)
 ├── WindowPositionStore              (geometry SSOT)
 ├── WindowDragBridge                 (sole mutation path)
 ├── WindowDragProvider
 └── WindowPositionProvider
              │
              ▼
FloatingWindowBridge                 (maps Store + State → Model[])
              │
              ▼
FloatingWindowLayer                  (render only)
              │
              ▼
FloatingWindow                       (title-bar Pointer → Drag API)
```

### Dual SSOT

| Dominio | Owner |
|---------|-------|
| Window lifecycle / registry | `WindowState` + `WindowAPI` (D55) |
| Window geometry (`x`/`y`) | `WindowPositionStore` (D57) |

### Pipeline certificado

```text
pointerdown → setPointerCapture → beginDrag
pointermove → updateDrag
pointerup → endDrag → releasePointerCapture
        ↓
WindowPositionStore
        ↓
FloatingWindowBridge → FloatingWindowModel[]
        ↓
FloatingWindowLayer → FloatingWindow
```

## 5. API Freeze respetada

### D55 (intacta)

`WindowAPI`: `create`, `register`, `unregister`, `activate`, `focus`, `minimize`, `restore`, `close`, `get`, `getAll`.  
Sin `moveWindow`, `setPosition`, `beginDrag`.

### D56 (intacta)

- `FloatingWindowModel`
- `FloatingWindowProps`
- `FloatingWindowLayerProps`

Sin callbacks de drag en props públicas.

### D57 (congelada — interna)

- `WindowPosition` = `{ x, y }` absolutas de layer
- `WindowDragAPI` = `beginDrag` / `updateDrag` / `endDrag` only
- Coordinate Space Freeze: sin scroll/zoom/matrices/offsets de dock
