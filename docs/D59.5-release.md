## 2. Objetivo alcanzado

Incorporar infraestructura de snap geométrico:

- Motor puro `applyMagneticSnap(geometry, targets, config) → geometry`
- Target providers extensibles (Dock reservado D61)
- Composition en Drag (D59.2) y Resize post-constraints (D59.3)
- Prioridad Workspace > Window · empates nearest → priority → stable order
- Determinismo e inmutabilidad certificados
- Zero UX visual · Zero cambios de APIs públicas

---

## 3. Arquitectura final certificada

### Unified geometry write-paths

```text
Drag:
  Pointer → Drag Math → Providers.collect → applyMagneticSnap → GeometryState

Resize:
  Pointer → Resize Math → GeometryConstraints → WorkspaceConstraints
        → Providers.collect → applyMagneticSnap → acceptSnapAxes → GeometryState
```

### Dependency direction (FROZEN)

```text
DragBridge ──► compose ──► Engine
ResizeBridge ──► compose ──► Engine
Providers ──► compose (targets)
Engine ──► ∅
```

### Host stack

```text
WindowManager
 ├── WindowState / WindowAPI          (D55 FROZEN)
 ├── WindowGeometryState              (geometry SSOT)
 ├── SnapConfig + SnapTargetProviders (Workspace + Window)
 ├── WindowDragBridge                 (move + snap composition)
 ├── WindowResizeBridge               (resize + constraints + snap)
 ├── WindowGeometryProvider
 ├── WindowDragProvider
 └── WindowResizeProvider
              │
FloatingWindowBridge → FloatingWindowLayer → FloatingWindow
```
