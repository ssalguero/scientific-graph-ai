## 6. LOC Methodology Snapshot

**Propósito:** congelar *cómo* se mide inventario de tamaño para la base D45–D63 — **sin recalcular métricas históricas** (D37 B-*) y **sin modificar código**.

| Campo | Valor congelado en D64.0 |
|-------|--------------------------|
| Unidad | Archivos fuente fundacionales `*.ts` / `*.tsx` |
| Alcance | Solo directorios listados en §3 (foundation surface) |
| Método | Enumeración read-only del filesystem (`Get-ChildItem` / equivalente) sobre rutas certificadas |
| Exclusiones | `node_modules`, tests fuera de foundation, `page.tsx` (orquestador producto), dominio científico |
| Recálculo LOC absoluto | **NO ejecutado en D64.0** — no se publican totales LOC nuevos; no se alteran baselines D37/D43 |
| Uso futuro | D64.x puede citar este método para inventarios; Amendment requerido para cambiar el método |

**Inventario de archivos (conteo de presencia, no LOC):** al momento D64.0 el árbol §3 contiene **~120** archivos `.ts`/`.tsx` en la superficie fundacional listada (ui lib + ui components + workspace + toolbar + inspector + docking + layout-engine + windows root + series + tabs + tab-ui + content).
