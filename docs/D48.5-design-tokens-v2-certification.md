## 6. Métricas

### 6.1 Archivos / consumidores (serie D48.2–D48.4)

| Métrica | Valor / nota |
|---------|----------------|
| Archivos núcleo tokens | `src/lib/ui/tokens.ts` · `theme.ts` · `index.ts` |
| Workspace contract | `WorkspaceTokens.ts` (delega) · Layout/Content usan `WORKSPACE_TOKENS` |
| Componentes Sidebar migrados | `Sidebar` · `SidebarItem` · `SidebarGroup` · `SidebarSection` |
| Layout UI migrados | `PanelHeader` · `PanelBody` · `PanelFooter` · `Divider` · `SectionTitle` |
| Buttons | `IconButton` → `UI_TOKENS.spacing`; variantes vía theme helpers (facade) |
| Validators (D48.4) | `validate-design-tokens-v2.ts` · `validate-v11-d48-gate.ts` · scripts npm |
| Ajuste D47 gate | `workspace.tokens.frozen.shape` alineado a SSOT D48.2 |

### 6.2 Duplicaciones eliminadas (D48.2)

| Antes | Después |
|-------|---------|
| `elevation` vs `shadows` literales duplicados | `elevation` deriva de `shadows` |
| `resultsSubsectionCard` ≡ `dataDatasetCard` | Composición compartida |
| Fragments border / transition / radius / shadow inlined en theme | Primitivos únicos en `tokens.ts` |
| Theme como segunda SSOT de class strings | Theme = facade sobre `UI_TOKENS` |

### 6.3 SSOT alcanzado

```text
UI_TOKENS          = única fuente de verdad (primitives + semantic compositions)
theme.ts           = facade pública (re-exports + helpers; sin redefinir primitivas)
WORKSPACE_TOKENS   = contrato público Workspace (shell / mainColumn / inner)
icons.ts           = registry congelado (sin cambio en D48)
```

- **`UI_TOKENS`** es la única fuente de verdad del sistema visual consolidado.
- **Theme** permanece como facade: API pública intacta; valores delegados a `UI_TOKENS`.
- **`WORKSPACE_TOKENS`** conserva su contrato público; Runtime Workspace no accede a categorías internas de `UI_TOKENS` salvo vía ese contrato.
- Sidebar / layout UI consumen `UI_TOKENS` tras D48.3 Wiring.
- Sin cambios visuales ni funcionales en la serie D48 (move-only / wiring / governance).
