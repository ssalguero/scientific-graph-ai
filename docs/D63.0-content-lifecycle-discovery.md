## 1. Executive Summary

**D63 — Lifecycle + Tab ↔ Series Wiring** mantiene el título certificado por D62.12 y entrega una arquitectura **híbrida**: infraestructura mínima para resolver `OpaqueContentHandle` → `ContentDefinition` registrada, más un bridge de mapeo Tab↔Series 1↔1.

D63 **no** mueve módulos científicos, **no** crea un workspace paralelo, **no** implementa lifecycle completo de WindowManager, **no** monta comportamiento en producto.

| Principio | Contrato D63 |
|-----------|--------------|
| Milestone certificado | Título = Lifecycle + Tab ↔ Series Wiring (sin reescribir NEXT D62.12) |
| Content opaco | `ContentDefinition = { id, kind, title }` — sin renderers |
| Registry SSOT | `ContentRegistry` única autoridad de catálogo |
| Dominios desacoplados | `ContentRegistry ≠ TabRegistry`; tabs solo conocen `contentId` |
| Resolve path | Solo `ContentBridge.resolve` → `ContentRegistry.get` (sin caché) |
| Host | Props-only · sin ownership · `null` / `children` |
| Tab↔Series | `TabSeriesBridge` 1↔1 · mapping/queries only |
| Ubicación | `windows/content/` · barrel propio · sin leak `windows/index.ts` |
| Producto | Zero UX / Zero Product Change · D63.8 library-only · never mounted by `page.tsx` |
| Compatibilidad | WindowAPI / Series / Tabs / tab-ui / Floating / Drag / Resize / Snap / D47 Workspace intactos |

```text
D63 = CONTENT RESOLUTION + TAB↔SERIES MAPPING (hybrid)
D63 ≠ scientific renderers (Graph / Table / Inspector / Heatmap / PCA)
D63 ≠ WindowManager product lifecycle (open/close tab flows)
D63 ≠ WorkspaceContent / workspace-content/ package
D63 ≠ N↔M Tab↔Series cardinality
D63 ≠ ContentBridge cache
D63 ≠ ContentHost ownership of Registry
D63 ≠ page.tsx mounting / visual changes
D63 ≠ extensión WindowAPI / campos content en Window*
```

Equivalente incremental:

```text
D55 → ventanas
D56 → floating
D57 → drag
D58 → resize
D59 → snap
D60 → series
D61 → tabs (infra)
D62 → tabs UI / policy / switch
D63 → content resolve + Tab↔Series mapping (hybrid)
D64+ → renderers + product lifecycle
```

## 6. API Freeze (LOCKED)

### 6.1 ContentDefinition — deliberadamente opaco

Exactamente tres campos. **Prohibidos en D63:** `component`, `factory`, `renderer`, `props`, `metadata`.

```ts
export type ContentDefinition = {
  id: string;      // equals OpaqueContentHandle.contentId
  kind: string;    // opaque infra kind — not scientific enum
  title: string;
};
```

### 6.2 ContentHostProps

```ts
export type ContentHostProps = {
  contentId: string;
};
```

### 6.3 ContentRegistry (SSOT)

```ts
export type ContentRegistry = {
  register(definition: ContentDefinition): void;
  unregister(id: string): void;
  get(id: string): ContentDefinition | undefined;
  list(): readonly ContentDefinition[];
};
```

| Regla | Contrato |
|-------|----------|
| SSOT | Única autoridad de catálogo de contenido |
| Independencia | Sin TabId · sin imports de tabs · sin awareness de tabs |
| Semántica prevista | Duplicate `register` = no-op · `unregister` safe si ausente · clone-on-read (como Series/Tabs) |

### 6.4 TabSeriesBridge — 1↔1 identity

Cardinality frozen:

```text
TabId → SeriesId     (at most one)
SeriesId → TabId     (at most one)
```

**Out of freeze:** `Tab → N Series`, `Series → N Tabs`.

Surface (mirror WindowSeriesBridge, Tab-scoped):

```ts
bind / unbind / hasTab / hasSeries /
getSeriesForTab / getTabForSeries / clear
```

**Side-effect ban:** no SeriesSelection · no TabSelection · no WindowSelection · no Activation · no Focus. Mapping + queries only.

### 6.5 ContentBridge

```ts
resolve(handle: OpaqueContentHandle): ContentDefinition | undefined
```

| Regla | Contrato |
|-------|----------|
| Path | `ContentRegistry.get(handle.contentId)` en **cada** `resolve` |
| Cache | **Prohibida** (`HR-no-content-cache`) |
| Storage | Bridge no posee storage propio |

### 6.6 ContentHost — no ownership

| Acción | Permitido |
|--------|-----------|
| Consumir props | Sí |
| Retornar `null` / `children` | Sí |
| Crear contenido | **No** |
| Registrar contenido | **No** |
| Destruir contenido | **No** |
| Mutar Registry | **No** |
| Charts / CSS / Tailwind / Context / hooks / stores | **No** |

### 6.7 Barrel Freeze (LOCKED)

| Regla | Contrato |
|-------|----------|
| Content público | Solo `src/components/windows/content/index.ts` |
| Deep imports | **Prohibidos** |
| `windows/index.ts` | **Sin** leaks Content* / TabSeries* |
| Imports | `OpaqueContentHandle` / `TabId` vía barrel `tabs/` · `SeriesId` vía barrel `series/` |
| Workspace | **Prohibido** importar `src/components/workspace/**` |

## 7. Hard Rules (LOCKED)

| ID | Regla |
|----|-------|
| `HR-content-barrel-only` | Public path = `content/index.ts` only |
| `HR-no-windows-barrel-leak` | No Content*/TabSeries* en `windows/index.ts` |
| `HR-opaque-handle-ssot` | Lookup key = `OpaqueContentHandle.contentId` ≡ `ContentDefinition.id` |
| `HR-content-def-opaque` | Definition = `{ id, kind, title }` only — no renderer fields |
| `HR-registry-ssot` | `ContentRegistry` es el único catálogo |
| `HR-registry-decoupled` | `ContentRegistry ≠ TabRegistry`; registry never knows tabs; tabs only know `contentId`; resolve only vía ContentBridge |
| `HR-no-content-cache` | `ContentBridge.resolve` siempre llama `ContentRegistry.get`; never caches results |
| `HR-host-no-ownership` | ContentHost does not create/register/destroy/mutate content or Registry — props only |
| `HR-no-scientific` | No Graph/Table/PCA/Heatmap/Inspector imports or kinds |
| `HR-no-product-wiring` | No `page.tsx`, WindowManager, FloatingWindow content swap |
| `HR-no-workspace-shell` | `windows/content/` imports nothing from `src/components/workspace/` |
| `HR-tab-series-1to1` | TabSeriesBridge cardinality = 1↔1 only |
| `HR-tab-series-mapping-only` | Mapping/queries only — no SeriesSelection / TabSelection / WindowSelection / Activation / Focus |
| `HR-lifecycle-deferred` | No WindowManager tab product lifecycle in D63 |
| `HR-d62-freezes-intact` | **Prohibido** romper API / Barrel / Governance D55–D62 |
| `HR-no-deep-imports` | **Prohibido** deep imports desde fuera de barrels |
| `HR-zero-product-default` | No exigir cambios de producto en `page.tsx` |
| `HR-d63-8-library-only` | Helper Switch→Bridge→Host = library-only; **never mounted by `page.tsx`** |

Estas Hard Rules son **irreversibles** a efectos de D63–D64+: dual-SSOT de content, cache en Bridge, ownership en Host, o N↔M prematuro crea deuda estructural.
